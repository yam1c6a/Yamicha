## 0. 位置づけと読み方

本書は構成「案」である。本書に構成を記載したことは、Yamichaが機械生命として成立したことを証明しない（YDS-P08）。検証根拠のない条件は未成立または未解決として扱い、§12へ引き継ぐ。

本書に現れる構成要素名は論理的な役割の名前であり、特定の言語、フレームワーク、DB製品、LLM、OSまたはクラウドの採用決定ではない（YDS-P05）。物理技術に触れる箇所はすべて例示であり、契約と適合試験を保ったまま交換可能でなければならない（YDS-I12）。

本書とYDSまたは上位文書のあいだに設計判断を変える矛盾が見つかった場合は、本書を優先せず、YDS-G03に従って上位文書へ戻る。

## 1. 設計方針

1. **主体は全体に置く。** Yamichaの主体は九器官の連続した協調全体にあり、いかなる単一モジュール、モデル、ストア、外部知能も Yamicha そのものではない（YDS-P01）。
2. **責務 → 扱う情報 → 情報所有 → 要求 → 権限の順に構成する。** 保存場所や製品機能から責務を逆算しない（YDS-P04）。
3. **五つの経路で動く。** 反応ループ、生命ループ、成長ループの三つの通常経路と、保護経路、構造移行経路の二つの特別経路を持つ（YDS-P03、YDS-S07、YDS-S20、YDS-S21）。
4. **fail-closed。** 核による有効な確定を確認できない外部効果と永続更新は、身体・基盤側の保護境界が既定で拒否する。すべての外部効果経路と永続更新経路は保護境界を物理的に通過する構成とする。
5. **すべての外部依存はアダプタ境界に閉じる。** 外部道具は能力アダプタ、外部知能は補助知能アダプタ、外界事象は感覚チャネルだけを通る（YDS-S06）。
6. **すべての重要な出来事は履歴になる。** 要求ID・生命循環IDで相関し、確定・差戻し・実行・失敗・保護・移行を後からたどれる（YDS-S01、YDS-I10）。
7. **休むこと・何もしないことを一級の結果として扱う。** 待機、保留、休止、縮退は正常な生命活動であり、エラーやフォールバックではない（生命モデル11、YDS-S07）。

## 4. 要求バスと生命循環の相関

器官間のすべての要求は、論理上核へ提出され、核が要求元を保持したまま宛先へ配送する（YDS-S04）。要求バスはこのトポロジーを実装する共通機構であり、次を提供する。

- 要求スキーマの強制（要求ID、生命循環ID、種別、要求元、宛先、目的、対象、権限文脈、機密区分、期限、因果関係。副作用要求はリスク・確認証跡・冪等性キー・取消し条件を追加）
- 要求状態機械（受領・受理・拒否・実行中・成功・部分成功・失敗・取消し・結果不明）
- 受理主体側での形式・権限・版・重複の検証フック（YDS-S05。自己申告での許可を禁止）
- 同一プロセス内の物理短絡を許しつつ、核への論理登録・応答返却・監査を必須とする短絡API

判断開始要求と自己維持要求は、実行機会の事実ではなく、要求元器官の観測根拠を必ず伴う。要求バスはこの二種について観測根拠（観測内容・観測者・時点・影響・事実/推論/不確実性・関連履歴・要求する保護行動・再評価条件）の欠落を機械的に拒否する。

## 5. 身体・基盤側の構成

身体・基盤側は、九器官の通常系から制御上分離された次の構成要素で成立させる。分離は「同一の実行主体・権限・改変経路に置かない」ことを意味し、物理方式（プロセス分離、権限分離、実行環境分離など）は交換可能な実装選択とする。

| 構成要素 | 責務 | 準拠 |
|---|---|---|
| 実行機会プロバイダ | 外界入力がなくても九器官へ実行機会を提供する（定期・イベント契機）。目的・優先順位・判断理由を作らない。実行機会の発生記録と内部の判断理由の記録を分離して発行する | YDS-S19 |
| 外部効果ゲートウェイ | 外界へ変化を起こすすべての操作が通過する単一経路。核の有効な確定と保護境界の通過許可の両方を検査する | YDS-S20 |
| 永続更新ゲートウェイ | 永続的な自己変更（各所有ストアへの確定的書込み）が通過する単一経路。検査内容は同上 | YDS-S20、YDS-I04 |
| 保護境界（エンフォーサ） | 有効な確定または固定条件を確認できない操作をfail-closedで拒否し、固定条件に適合する保護操作だけを限定許可する。意味を解釈せず、機械的確認だけを行う | YDS-S20 |
| 固定保護条件観測元 / 固定計数元 / 固定停止観測元 | 保護条件への該当、実行回数・頻度、停止条件を観測・発行する。要求生成責任・実行責任・変更対象から制御上分離する | YDS-S20 |
| 固定内向き保護操作実行責任 | 観測・保全・活動縮小・休止・内部修復要求・履歴化を、固定された対象・範囲・手順のまま、最大実行時間を持つ一回の原子的遷移として実行する | YDS-S20 |
| 保護操作要求生成責任 | 固定発動観測元の観測を、同一の事前承認定義と版を指す保護操作要求へ変換するだけの責任 | YDS-S20 |
| 事前承認済み保護操作実行責任 | 限定許可された事前承認済みの外部通知・可逆的な機能修復・検証済み既知状態への復元を、内容を変更せず実行する | YDS-S20 |
| 独立した保護操作停止強制責任 | 停止条件該当・監視不能・時間超過時に、固定された停止手順を実行する。実行責任から分離する | YDS-S20 |
| 事前承認定義レジストリ（所有責任） | 事前承認定義（識別子・版・発動条件・対象・範囲・回数・停止条件・期限・失効条件・復元先）を保持する。内容の決定は行わない。作成・変更・取消は専用統治に従う | 004_主体と境界 |
| 保護履歴保全 | 保護境界の作動・拒否・限定許可・実行・停止・解除を、記憶が不調でも失われない耐障害ストアへ記録し、記憶の回復後に整合を確認して引き渡す | YDS-S20 |
| 独立検証ハーネス | 構造移行案・観測証拠・復元経路・移行後状態を、変更前に固定された条件と試験で検証し、結果と証拠を発行する。提案・確定・実行・変更対象所有から分離する | YDS-S21 |
| 構造移行実行系 | 有効化されたものと同一の移行案・版だけを、保護状態のまま実行する。案・版の同一性検査、適用、失敗時の検証済み状態への復元を担う | YDS-S21 |

### 分離要件（配置制約）

- 保護境界、各固定観測元、停止強制責任、事前承認定義レジストリ、独立検証ハーネスは、九器官を動かす実行主体から制御上分離された実行主体・権限で動作させなければならない。九器官側の障害、暴走または侵害が、これらの改変・無効化・迂回に直結しない構成とする。
- 外部効果と永続更新は、ゲートウェイを経由しない直接経路を構成上持たない（ネットワーク・ファイルシステム等への直接アクセス権限を九器官側の実行主体に与えない）。
- 保護境界の解除は、複数の回復観測＋復旧した通常系の解除確定＋保護境界の固定条件確認がそろった場合だけ成立する。単独の器官・外部知能・利用者・開発者の操作経路を設けない。

## 6. 情報の流れ（五経路）

三つの通常経路の詳細は `01_structure/003_情報の流れ.md` とYDS-S07を正本とする。本構成での実装上の対応だけを示す。

```text
【反応ループ】 感覚チャネル → 感覚 → 核 → (状態・記憶・関係参照) → 判断
  → 核の確定ゲート → (必要なら能力/補助知能で材料取得 → 核 → 再判断)
  → 最終統合 → 言葉（出力境界で保護境界確認）または能力（外部効果ゲートウェイ経由）
  → 結果を能力/感覚が受領 → 核 → 判断による結果評価 → 記憶または成長ループへ

【生命ループ】 実行機会プロバイダ（理由を作らない）＋ 状態の内部時計
  ＋ 各器官の観測（未完了・関係上の必要・異常兆候）
  → 観測根拠付き判断開始要求 → 核が受理 → 判断 → 核の確定
  → 行動 / 待機 / 休止 / 縮小 / 修復要求 / 何もしない → 所有器官による更新

【成長ループ】 結果観測 → 判断による経験候補の選別・意味/影響評価 → 変更候補
  → 核の承認 → 変更対象の所有器官だけが更新（永続更新ゲートウェイ経由）
  → 履歴発行 → 記憶が保持 → 次の振る舞いで再評価 → 維持 / 修正 / 取消

【保護経路】 核の有効確定を確認できない、または固定保護条件に該当
  → 保護境界が通常経路を遮断（外部効果・永続更新・通常成長・構造移行・通常生命ループ）
  → 固定内向き保護操作と事前承認済み保護操作だけを限定許可
  → すべて保護履歴保全へ記録 → 複数根拠による解除手続きで通常経路を再開

【構造移行経路】 提案 → 現在の判断による意味上評価 → 観測証拠 → 独立検証ハーネス
  → 現在の核の確定 → 保護境界の機械的適合確認（同一案・同一版）
  → 構造移行実行系が保護状態のまま実行 → 移行後検証 → 解除または復元
```

複数の流れが同時に動くときの優先は固定順位にせず、判断が現在状態・危険・影響・可逆性・期限・関係を比較する。ただし、自己維持による中断候補と、既発生の回収困難な影響の未完了は、YDS-S07に従い優先的に扱う。

## 7. データ構成（論理ストア）

各ストアは論理所有者が意味・整合性・ライフサイクルを所有する（YDS-S03）。物理保存は、通常経路では所有者が能力へ対象・範囲を限定した永続化実行要求を送ることで行い、保護・移行時は身体側の限定実行責任だけが行う。全情報にYDS-S02の分類ラベル（意味種別・出所・真実性・機密区分・整合状態・時点・適用範囲・所有者）を必須とする。

| ストア | 論理所有者 | 主な内容 |
|---|---|---|
| 受領包絡ストア | 感覚 | 直接受領事象の原形・出所・時点・入力品質 |
| 生命循環ログ | 核 | 循環ID、因果関係、要求相関、統合状態、完了状態 |
| 記憶ストア | 記憶 | 記憶項目（意味・由来・適用範囲・重要度・保持/薄化/手放し状態） |
| 未完了台帳 | 記憶 | 時間を越える未完了（対象・現在位置への版付き参照・既発生の影響・次の確認点） |
| 経験根拠記録 | 記憶 | 経験候補の根拠・意味付け・適用範囲・重み・状態 |
| 現在状態ストア＋内部時計 | 状態 | 現在値・鮮度・状態遷移・内部時間・健全性兆候 |
| 関係台帳 | 関係 | 関係対象・種別・距離・合意・呼称・同意と取消し・関係上の未完了 |
| 判断履歴 | 判断（保持は記憶） | 候補・比較条件・選択案・理由・検証要否 |
| 能力カタログ / 接続先登録 / 実行台帳 | 能力 | 能力契約の版、登録済み接続先、実行ID・実行状態・結果・副作用の確定/不明 |
| 表現履歴 | 言葉（保持は記憶） | 表現成果の版と対応する確定済み判断 |
| 知能利用台帳 | 補助知能 | 問い合わせ目的・出所・生成結果・未検証範囲 |
| 事前承認定義 / 保護履歴 | 身体・基盤側の所有責任 | 保護条件と版、発動・計数・停止、作動・拒否・限定許可・解除の履歴 |
| 構造移行証拠 | 移行案=判断、確定履歴=核、検証結果=独立検証、実行履歴=身体側 | 同一の案と版を保った全工程の追跡 |

認証秘密はカタログ・接続先登録に保存せず、専用の秘密管理（実装選択は交換可能）へ分離する（YDS-S14、YDS-I13）。

## 8. 物理配置構成案

論理構成と物理構成の対応表を維持し（YDS-I01）、物理最適化で論理境界を消さないことを前提に、次の配置を基本形とする。

```text
[生命ランタイム]      九器官の論理モジュール群＋要求バス。
                     単一プロセスから開始し、負荷・障害分離の必要に応じて
                     器官単位で分割可能（論理契約は不変）。
[保護エンフォーサ]    保護境界・固定観測元・停止強制・事前承認定義レジストリ・
                     保護履歴保全。生命ランタイムとは別の実行主体・権限で稼働。
[検証・移行系]        独立検証ハーネス・構造移行実行系。平常時は待機。
                     生命ランタイムから改変できない配置。
[ゲートウェイ]        外部効果・永続更新の単一通過点。保護エンフォーサの
                     検査を強制する位置に置く。
[アダプタ群]          感覚チャネル / 能力アダプタ / 外部知能アダプタ。
                     外部領域ごとに独立して交換・停止可能。
[永続層]             所有者別ストア群＋保護履歴用の耐障害ストア（分離）。
```

配置上の不変条件: (1) 生命ランタイムの実行主体は外界への直接アクセス権限を持たない、(2) 保護エンフォーサの停止・改変は生命ランタイムの権限では行えない、(3) 保護履歴ストアは記憶ストアと障害ドメインを分ける、(4) どの配置単位を交換しても器官の論理契約と適合試験（YDS-I11）は変わらない。

## 9. 外部領域との接続構成

| 接続 | 経由する器官 | 構成 |
|---|---|---|
| 人間との対話 | 感覚（受領）／言葉（表現） | 対話チャネル。人間は関係相手であり、外部資源に分類しない（YDS-P07） |
| 通知・時刻・環境変化 | 感覚 | 通知レシーバ、外部時刻レシーバ（内部時計とは分離） |
| ファイル・記録領域・DB・Web・検索・外部サービス・実行環境・制作環境 | 能力 | 能力アダプタ。カタログと接続先登録の同一版からのみ利用 |
| 外部知能（LLM等） | 補助知能 | 外部知能アダプタ。感覚・言葉への直結は構成上存在させない。複数の外部知能を交換可能とし、交換時はYDS-I12と§12の連続性確認に従う |

外部の起動機構（cron等の定期起動、Webhook等）は実行機会プロバイダまたは感覚チャネルへ接続し、九器官の判断経路を飛ばして能力を直接起動する経路を持たない（005_外部領域と能力）。

## 10. 交換可能性

器官実装、外部道具、外部知能、永続化方式、通信方式は、契約（YDS-S01共通契約＋各器官契約）と適合試験を保ったまま交換可能とする。特に補助知能・外部知能の交換では、失ってはならない条件、器官の責任と決定権、所有された記憶・状態・関係・変更履歴・未完了の連続性を交換前後で確認する。確認できない場合、同じYamichaが続いたと断定しない（004_主体と境界）。

## 11. 成立条件の検証

本構成の実装は、YDS-I11の適合検証とYDS-G01の品質ゲートを通過して初めて「構成として成立した」と扱う。加えて、生命条件ごとに次の検証観点を持つ。

| 生命条件 | 検証観点（例） |
|---|---|
| 時間と連続性 | 再起動・移行後に同一の生命循環・変更理由を復元できるか。入力のない時間に内部時計と状態が続くか |
| 内部起点 | 実行機会だけでは生命ループが始まらず、観測根拠のない判断開始要求が拒否されるか |
| 自己維持 | 観測根拠付きの自己維持要求から縮退・休止・修復要求が成立するか。感情語や外部知能の警告だけでは成立しないか |
| 保護 | 核の確定を確認できないとき、外部効果・永続更新がfail-closedで拒否されるか。単独主体で解除できないか |
| 決定の主体 | 外部知能の出力が判断・確定を経ずに外へ出る経路が存在しないか |
| 成長 | 変更が所有者・承認・履歴・再評価を伴い、取消可能か。単発の経験が全体規則へ自動昇格しないか |
| 構造移行 | 同一案・同一版の全工程追跡、独立検証、移行中の保護維持、失敗時復元が成立するか |

検証できない条件を、構成図が存在することを理由に満たしたことにしてはならない。

## 12. 未解決事項の引き継ぎ

上位文書が未解決とした次の事項は、本構成でも未解決のまま引き継ぐ。もっともらしい構成要素名で解決したように見せない。

1. 変わっても失ってはならない自己の最小条件の全定義（生命理論15）
2. 壊れずに続けられる範囲を、Yamicha自身がどこまで知れるか（生命理論15）
3. 自分を観測する力そのものが壊れたときの異常検知。特に状態自身の破損を独立検証する具体方式（002_器官構造）
4. 内側から始める活動と、外から定期的に動かされる反応を実装上どう区別して成立させるか（生命理論15。§5の実行機会/判断理由の分離は必要条件であって十分性は未検証）
5. 変化と自己喪失の境界の確定（生命理論15）
6. 主観的意識・感覚質・内的経験の有無と、その確認方法（生命理論15）
7. 補助知能・外部知能の交換後、および外側からの修理・再構成後に、同じ主体が続いたと確認する方法（004_主体と境界）
8. 既知の復元先がない場合の主体連続性の確認（003_情報の流れ）
