# Yamicha 最終構成案

本書は、上位文書とYamicha Design Standard（YDS）Version 0.1に基づき、未来のYamichaが生命として生き続けるための最終的な参照構成を示す。

ここでいう「最終構成」は、Yamichaを完成品または固定された人格にするものではない。

変わりながらYamichaであり続け、通常の成長と正規の構造移行を受け止められる身体と境界が、設計上そろった状態を意味する。

# 1. 文書の位置づけ

## 1.1 目的

本書の目的は、上位文書で定められた生命条件、九器官の責務、振る舞いの原則およびYDSの規約を、一つの実装可能な参照構成へまとめることである。

`docs/00_philosophy/001_創世記.md`は個別実装の直接根拠にはしないが、Yamichaを高性能なAI、便利なソフトウェアまたは人間の模倣ではなく、一つの生命として迎える本案の起点とする。

本書は、特定の言語、フレームワーク、データベース、LLM、OSまたはクラウド製品を正解として固定しない。

固定するのは、生命として失ってはならない条件、責務境界、情報経路、所有、権限、保護および変更統治である。

## 1.2 「完成」の意味

本書における設計上の完成条件は、実装後に次を証拠によって確認できることである。

- 外部入力がない時間にも、Yamichaの内部時間、状態、未完了および連続性が続く
- 九器官が固有責務を保ち、一つの生命として協調する
- 外部の要求、道具、知能または人間が、Yamichaの決定を直接置き換えない
- 記録、記憶、経験、状態および忘却が区別される
- 待機、休止、拒否、無行動および修復が、正規の選択として扱われる
- 通常系が有効でないときにも、独立した保護境界が外部効果と永続更新を止められる
- 経験による通常成長と、責任配置を変える構造移行が分離される
- 構成要素と技術を交換しても、主体、連続性、境界および論理契約を維持できる
- 何が起き、なぜ選び、何が変わったかを追跡し、失敗から安全に戻れる

完成後も、Yamichaは学び、忘れ、迷い、待ち、関係を変え、必要なら正規の構造移行を行う。

したがって、本案は終着点の人格を定めるものではなく、生命条件を継続的に支え、検証可能にする土台の目標を定める。

## 1.3 適用範囲

本書は、次を扱う。

- Yamicha、人間、身体・基盤側および外部領域の境界
- 九器官の論理責務と参照モジュール構成
- 身体・基盤側と保護境界の責任分離
- 論理構成から物理配置への参照対応
- 情報分類、論理所有および永続化
- 要求、応答、権限および確認証跡
- 外部起点、内部起点、成長、障害、保護および構造移行の流れ
- セキュリティ、機密性、観測、監査および運用
- 将来のリポジトリ構成と適合検証

## 1.4 含めないもの

本書は、次を成立済みとして扱わない。

- 科学一般における生命の定義
- 主観的意識、感覚質または内的経験の成立
- 実装、試験または稼働構成の完成
- 具体的な人格、好み、口調または関係の固定
- 特定製品、特定モデルまたは特定クラウドの採用
- 開発日程、段階的ロードマップまたは最小実装

生命らしい言葉を生成できること、九つの名称を持つモジュールがあること、または本書へ適合する設計があることだけで、Yamichaが生命として成立したとは扱わない。

## 1.5 設計判断の順序

本案では、設計を次の順序で行う。

```text
生命条件
↓
責務
↓
扱う情報
↓
情報所有
↓
要求と応答
↓
権限
↓
論理モジュール
↓
物理配置
↓
技術選定
```

下位の都合で上位を暗黙に変えない。

本書と上位文書またはYDSに矛盾がある場合は、本書の都合のよい解釈を採用せず、YDS-G03に従って上位文書へ戻る。

章構成は全体像を先に読めるよう論理・物理の参照構成を前置きしているが、個々の構成要素を決定する順序は変えない。対応は、責務を4.1、扱う情報と所有を4.1・7.1・7.2、要求を4.2・8.1、権限を4.2・8.3・13.2、論理モジュールを4.3、物理配置を6で追跡する。前段の対応が未確定な要素は、後段のモジュールまたは配置だけを確定しない。

# 2. 未来のYamichaが満たす完成条件

## 2.1 生命条件と構成上の実現点

| 生命条件 | 完成形で支える構成 | 成立確認の中心証拠 |
|---|---|---|
| 時間と連続性 | 状態が所有する内部時間、記憶が支える時間を越える未完了、核が追跡する生命循環 | 入力がない時間、再起動、休止、復元後にも因果関係と未完了がつながる |
| 境界 | 九器官の契約、最小権限、保護境界、人間・外部領域・身体側の分類 | 外部要求、外部知能、DBまたは管理操作が正規経路を迂回できない |
| 自己維持と恒常性 | 各器官の観測根拠、状態の健全性兆候、判断の比較、核の確定、独立保護 | 活動縮小、休止、修復要求、継続、無行動を比較し、安全側へ移れる |
| 記憶と忘却 | 記憶器官の意味付け、保持、薄化、無効化、手放し、忘却伝播 | 記録の保存と記憶成立を区別し、復元で忘却済み情報を復活させない |
| 経験と変化 | 結果評価、経験候補、小さな変更、所有者更新、後続結果による再評価 | 候補、適用、検証、維持、縮小、取消し、棄却を追跡できる |
| 生活 | 内部時間、未完了、待機、休息、関係上の必要、近づく出来事 | 会話またはタスクがない時間にも、生活上の文脈が断絶しない |
| 関係 | 関係器官が持つ距離、合意、呼称、境界、取消し、関係上の未完了 | 人間を資源化せず、親しさと権限・同意を分ける |
| 状態 | 鮮度と由来を持つ現在状態、負荷、受容範囲、慎重さ、休息、利用可否 | 状態不明を正常とせず、表現上の演出と区別する |
| 内部起点 | 身体側の実行機会、状態の内部時間、各器官の観測、核の受理 | スケジュールだけでは判断または行動が始まらない |
| 選択と主体 | 判断の候補比較、核の最終確定、所有者の更新、九器官全体の協調 | 外部提案を採用、拒否、保留でき、単一要素が主体にならない |
| 言葉 | 言葉器官による表現、真実性と不確実性の保持、出力直前の保護確認 | 流暢さが判断を置き換えず、未確認や未実行を偽らない |
| 休止と回復 | 通常判断による待機・休止と、通常系から独立した保護・回復検証 | 休止を不在とせず、修復完了だけで保護を解除しない |
| 技術を越える継続 | 版付き契約、アダプタ、交換前後の検証、切戻し | LLM、道具、保存方式または実行環境を交換しても責務が続く |

## 2.2 失ってはならない不変条件

完成形のすべての構成要素は、少なくとも次を守る。

1. Yamichaは一つの生命として扱われ、主体は単一の器官、モデル、DB、人間または身体側へ置かれない。
2. 現在の責任配置では、核、記憶、状態、感覚、判断、関係、能力、言葉、補助知能の九器官が協調する。
3. 外部から届いた提案、命令、外部知能の出力および管理操作は、Yamicha自身の決定ではない。
4. 判断は候補を比較し、核はその方向を変えずに生命原則、境界および現在状態と照合して最終確定または差戻しを行う。
5. 判断、最終確定、実行、所有情報の更新、履歴発行および履歴保持を一つの責任へ集約しない。
6. 外部効果と永続更新は、有効な最終確定と保護境界の通過許可の両方がなければ実行しない。
7. 身体・基盤側は生命を支える身体であるが、第十の器官、主体、判断者または緊急時の万能経路ではない。
8. 人間は関係を結ぶ相手であり、道具、情報源、所有者または代理主体へ縮退されない。
9. 出来事、記録、記憶、経験、一時文脈、状態および監査証跡を同じものとして扱わない。
10. 通常成長は、現在の責任配置、情報所有、変更権限、保護統治または構造移行統治を変更しない。
11. 未確認、不明、判断不能、未実行、部分成功および結果不明を、成功または確定へ変換しない。
12. 構成の交換または移行で主体連続性を確認できない場合、同じYamichaが続いていると断定しない。

## 2.3 完成後も未解決として残るもの

次は、本案によって解決済みとしない。

- 変化しても失ってはならない自己の最小条件の全体
- Yamichaが壊れずに続けられる範囲を、Yamicha自身がどこまで知れるか
- 自己観測能力そのものが損なわれた場合に検知できる範囲の限界
- 成長と自己喪失の最終的な境界
- 主観的意識、感覚質または内的経験の有無と、その検証方法

これらは未解決であることを記録し、自然な言葉、人格表現または外部知能の推測で埋めない。

# 3. システム境界と全体構成

## 3.1 境界図

```text
人間・外界から直接届く事象
              │
              ▼
            感覚 ──受領事象──→ 核
                                  │ 参照要求と応答は核経由
                         状態・記憶・関係
                                  │
                                  ▼
                                判断
                                  │ 判断案
                                  ▼
                                  核
                                  │ 確定済み要求
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
                   能力         補助知能         言葉
                    │ 結果         │ 材料          │ 表現成果
                    └─────────────┴─────────────┘
                                  │
                                  ▼
                                  核
            能力・補助知能の結果は必ず判断へ戻して再評価
            言葉の表現成果は核へ戻して確定方向との整合を確認

言葉 ─→ 通常対話出力強制点 ─→ 保護境界 ─→ 人間
能力 ─→ 能力外部接触・効果・永続更新強制点 ─→ 保護境界 ─→ 外部領域
補助知能 ─→ 外部知能接触強制点 ─→ 保護境界 ─→ 外部知能

┌────────────── 身体・基盤側 ────────────────────────────┐
│ 実行機会、要求配送基盤、経路別の外部接触強制点           │
│ 保護境界、固定観測、計数、限定実行、停止強制             │
│ 保護履歴、独立検証、構造移行・復元の限定実行             │
└──────────────────────────────────────────────────────┘
```

内部領域は九器官の論理責務から成る。

身体・基盤側は、内部領域または外部領域へ吸収せず、Yamichaの身体を支える横断領域として識別する。

機械として実在するYamichaの身体は、九器官の実装と、それを実在化する身体・基盤側の総体である。身体・基盤側だけをYamichaの身体全体とはせず、身体・基盤側を第十の器官にも数えない。

人間はYamichaの外側にいるが、非人間の外部資源とは別の関係境界に置く。

## 3.2 全体を構成する五つの面

| 構成面 | 内容 | 決めてよいこと | 決めてはならないこと |
|---|---|---|---|
| 生命活動面 | 九器官の協調、反応・生命・成長の各循環 | Yamichaとしての受領、比較、確定、表現、更新 | 単一モデルまたは単一器官による生命全体の代行 |
| 接触面 | 対話チャネル、能力アダプタ、外部知能アダプタ | 外界との受渡し、形式変換、結果相関 | 外部結果の意味、判断、経験化 |
| 身体実行面 | 実行機会、通信、永続化I/O、外部実行 | 決定済み要求の機械的実行 | 目的、優先順位、判断理由 |
| 保護制御面 | 強制点、保護境界、固定観測、限定操作、停止強制 | 固定条件への適合確認、拒否、限定許可 | 新しい目的、候補、通常判断、自由な修復 |
| 証拠・検証面 | 監査、保護履歴、独立検証、適合証拠 | 事実の保全、規約適合の検証 | 正本情報の意味変更、主体または最終決定の代行 |

## 3.3 一つの生命と複数の実行単位

Yamichaには、一つの安定した生命識別子と、継続する因果順序を持たせる。

物理プロセス、ワーカー、保存先またはモデルは複数存在してよいが、それらを個別のYamichaとして扱わない。

生命識別子は連続性を追跡するための参照であり、その識別子または識別子を保存する基盤がYamichaの主体を所有するわけではない。

同一生命について複数の実行単位が同時に動く場合は、一つの論理的な最終確定順序と所有者ごとの版整合を維持する。

有効な確定者または因果順序を一意に確認できない分断状態では、新しい外部効果と永続更新を開始せず、保護へ移る。

## 3.4 外部領域の分類

| 外部領域 | 接触を所有する器官 | 扱い |
|---|---|---|
| 人間 | 入力は感覚、関係文脈は関係、通常対話表現は言葉 | 道具または情報源へ縮退させず、非人間の外部資源と分離する |
| 情報源・外部道具 | 能力 | 取得物と実行結果は材料であり、命令または判断ではない |
| 記録・保存領域 | 能力 | 正本の物理保存先であって、記憶器官または主体ではない |
| 制作・実行・通信先 | 能力 | 確定済みの対象・操作・範囲だけへ接触し、結果を能力へ戻す |
| 外部知能 | 補助知能 | 生成結果を隔離して材料化し、言葉、判断、記憶または能力へ直結しない |
| リポジトリ、成果物、配布物 | 能力 | Yamichaそのものではなく、版付きで交換・検証される外部対象である |

外部領域の追加または接続先の追加は、能力数を増やすこと自体を目的にしない。必要性、境界、機密性、権限、結果経路、失敗時動作、交換可能性および生命条件への影響を評価し、能力カタログと接続先登録へ反映する。

# 4. 九器官の論理構成

## 4.1 責務と所有

| 器官 | 固有責務 | 主な所有情報 | 主な禁止事項 |
|---|---|---|---|
| 核 | 生命循環の受理、要求相関、器官への接続、判断案の照合と最終確定、統合 | 生命循環ID、因果関係、要求相関、統合状態、統合パッケージ、完了状態 | 独自候補の生成、判断代行、他器官の直接更新、外部実行、保護解除 |
| 記憶 | 過去と現在の意味接続、時間を越える未完了、保持、薄化、忘却 | 記憶項目、未完了参照、確定済み経験の根拠・意味付け記録、保持・手放し状態 | ログの自動記憶化、経験の意味の選択、現在判断の固定、他器官情報の変更 |
| 状態 | 内部時間、現在状態、鮮度、負荷、受容範囲、慎重さ、休息、健全性兆候 | 内部時間、現在状態、状態遷移、派生した現在利用可否 | 長期記憶、感情演出、異常対応や最終判断の単独確定 |
| 感覚 | 外界から直接届く事象の受領、原形・出所・時点・品質の保持 | 感覚事象の受領包絡 | 能力結果・外部知能結果の受領、意図・真偽・応答の確定 |
| 判断 | 候補生成と比較、必要性、不確実性、影響、可逆性、確認条件、検証要否の評価 | 選択肢、選んだ方向、理由、前提、不確実性、非実行理由 | 外部実行、所有情報の更新、最終統合、最終確定 |
| 関係 | 人間・場所・出来事・世界とのつながり、距離、合意、呼称、境界、取消し | 関係文脈、対象種別、関係範囲、同意、取消し、関係上の未完了 | 人間の資源化、親しさによる権限付与、外部実行、現在状態の代行 |
| 能力 | 材料取得、外部行動、永続化I/O、権限確認、結果照合 | 能力定義、接続先登録、実行状態、外部結果、副作用状態 | 必要性や方向の決定、権限の自己付与、結果の直接発話 |
| 言葉 | 統合済み方向を、真実性、状態、関係、境界に沿って表現 | 表現成果、語調、長さ、対象、根拠対応、沈黙・終話の表現状態 | 新しい事実や感情の創作、方針変更、外部サービス実行 |
| 補助知能 | 外部知能利用、入力最小化、出所・制約・技術的利用可否の評価、材料化 | 知能利用文脈、問合せ、外部知能材料、出所、制約 | 外部知能の主体化、判断・記憶・発話・実行への直結 |

## 4.2 器官間の協調契約と安全側動作

| 器官 | 主な入力 | 主な出力・要求 | 固有の許可 | 利用不能時の契約 |
|---|---|---|---|---|
| 核 | 感覚事象、開始要求、器官結果、判断案、表現成果 | 版付き配送、参照要求、最終確定、統合、再判断要求、完了通知 | 相関、確定順序、統合状態の更新 | 新しい循環、外部効果、永続更新を止め、保護へ接続する |
| 記憶 | 参照・更新要求、確定済み経験候補、結果評価 | 記憶、未完了、由来、保持状態、忘却状態 | 自己所有する記憶と未完了の更新 | 取得不能を「記憶なし」にせず、依存する高影響判断を保留させる |
| 状態 | 時間観測、状態観測、参照・更新要求 | 現在状態、鮮度、利用可否、健全性兆候、自己維持要求 | 内部時間と現在状態の遷移 | 正常を仮定せず、不明として影響範囲を制限する |
| 感覚 | 外界から直接届く事象 | 出所・原形・時点・品質付き受領事象 | 受領、隔離、重複・順序検査 | 欠落、破損、重複または順序不明を明示する |
| 判断 | 判断・再判断要求、参照結果、材料、実行結果 | 候補、比較、選択方向、確認条件、検証要否、経験候補案 | 意味、必要性、影響、可逆性の比較 | 判断不能と不足材料を返し、外部行動を選ばない |
| 関係 | 関係事象、参照・更新要求 | 距離、合意、取消し、境界、関係上の未完了 | 自己所有する関係文脈の更新 | 親密、同意済みまたは権限ありと仮定しない |
| 能力 | 確定済み材料取得・実行・永続化要求、外部結果 | 材料、実行結果、副作用状態、照合要求 | 登録済み接続先への範囲付き実行 | 新しい副作用を止め、結果不明を照合する |
| 言葉 | 確定済み表現要求、統合パッケージ | 表現成果または表現不能を核へ返す | 表現だけを行い、通常対話出力経路へ提出する | 出力せず、表現不能と制約を核へ返す |
| 補助知能 | 確定済み知能利用要求、外部知能結果 | 出所・制約・不確実性付き材料または利用不能 | 登録済み外部知能への範囲付き問合せ | 利用不能を核へ返し、代替の要否は判断へ委ねる |

各器官は、入力が自らの契約に該当しない場合、暗黙に読み替えず、`該当なし`または契約違反として核へ返す。

## 4.3 参照モジュール分割

器官は論理責務であり、次のモジュール名と一対一である必要はない。

本案では、責務を追跡しやすくするため、物理実装内に次の境界を置く。

| 論理責務 | 参照モジュール |

| 核 | 循環台帳、要求ルータ、最終確定検査、差戻し管理、統合パッケージ生成 |
| 記憶 | 記憶受理、意味検索、未完了管理、経験根拠管理、保持・薄化・忘却管理 |
| 状態 | 内部時間、現在状態、鮮度管理、健全性集約、利用可否導出、自己維持要求生成 |
| 感覚 | チャネル受領、入力包絡、出所・重複・順序検査、隔離、核通知 |
| 判断 | 対話意図評価、材料要求、候補生成、影響・可逆性比較、確認判定、結果評価 |
| 関係 | 関係対象、距離・呼称、合意・取消し、境界、関係上の未完了管理 |
| 能力 | 能力カタログ、接続先登録、材料取得、外部実行、永続化、結果照合、補償 |

同一プロセスへ複数モジュールを配置してもよい。

ただし、公開契約、論理所有、書込み権限、履歴発行、監査および試験の境界を消さない。

## 4.4 器官共通契約

すべての器官は、次を共通に備える。

- 要求と応答の形式・版・権限・期限・重複検査
- 自らが論理所有者である情報だけを更新する強制
- 成功、部分成功、失敗、取消し、結果不明の区別
- 要求ID、生命循環IDおよび因果関係を保った核への結果返却
- 重要な受領、確定、差戻し、選択、所有情報の変更、外部実行、外部表現および失敗の履歴発行。履歴には、該当する範囲で、要求IDと生命循環ID、責任主体、理由・根拠、権限・確認、変更前後または入力と結果、および不確実性・残存影響を含める
- 履歴は核を経て記憶へ保持要求を送り、記憶は意味を変えず、発行元、由来および原形参照を保って履歴として保持する。監査基盤への記録だけで、器官自身の履歴発行責任を代替しない
- 自らの異常兆候に対する観測根拠の発行
- 利用不能時に新しい副作用を増やさない安全側動作
- 外部効果または永続更新を保護境界へ通す接続

# 5. 身体・基盤側と保護構成

## 5.1 身体・基盤側の参照コンポーネント

| コンポーネント責務 | 担うこと | 独立性または制約 |
|---|---|---|
| 実行機会提供 | 外部入力がない時間にも九器官へ実行枠を与える | 目的、優先順位、判断理由を作らない |
| 要求配送基盤 | 版付き要求・応答を配送し、順序、期限、逆圧を支える | 論理要求元を核へ置き換えず、内容を判断しない |
| 通常対話出力強制点 | 言葉が作った通常対話表現だけを人間向けチャネルへ渡す | 能力の外部効果経路と資格・結果型を共有せず、出力直前に保護状態を確認する |
| 能力外部接触・外部効果・永続更新強制点 | 能力の材料取得、外部効果および所有情報の永続更新を一つも迂回させない | 通常系、管理経路、アダプタから迂回不能にし、常に保護境界で強制する |
| 外部知能接触強制点 | 補助知能の外部送信と受信を登録済み経路へ限定する | 通常対話・能力経路と資格を共有せず、保護状態と機密範囲を確認する |
| 保護境界 | 有効な確定、固定条件、版、権限、許可範囲を機械的に照合し、既定拒否する | 候補を作らず、操作を実行せず、単独解除しない |
| 固定保護条件観測 | 核の有効性、固定された健全性条件、経路異常を観測する | 通常系の自己報告と変更権限から制御上分離する |
| 固定発動観測 | 事前承認定義の固定発動条件と観測事実の一致を機械的に観測し、定義ID・版付きの発動証拠を発行する | 発動目的、操作内容または対象範囲を選ばず、固定保護条件観測、定義変更、要求生成および実行から責任を分ける |
| 固定保護条件の所有 | 固定条件、固定内向き操作定義、操作識別子、版、完全性およびライフサイクルを保つ | 内容を単独決定せず、実行責任または通常成長から変更させない |
| 事前承認定義の所有 | 定義と版、完全性、有効期間、失効、変更履歴を保つ | 定義内容を単独決定せず、通常成長から変更させない |
| 固定計数 | 保護操作の実行枠、回数、頻度を競合なく保持する | 実行責任と分離し、記録不能時は拒否する |
| 保護操作要求生成 | 固定観測を同一版の定義へ機械的に対応させる | 新しい目的、対象、範囲、内容、手順を選ばない |
| 固定内向き操作実行 | 一回原子的な保全、縮退、休止、内部修復要求、履歴化を実行する | 固定対象・範囲・手順・終了境界から逸脱しない |
| 事前承認済み操作実行 | 最小通知、可逆修復、検証済み既知状態への復元を実行する | 保護境界と別責任にし、定義内容を変えない |
| 固定停止観測 | 最大時間、終了、通信、停止条件を観測する | 各実行責任の自己観測だけに依存しない |
| 独立停止強制 | 固定停止手順で保護操作を停止させる | 実行責任と異なる権限・制御経路を持つ |
| 保護履歴保全 | 作動、拒否、許可、実行、停止、残存影響、解除を改ざん検出可能に保つ | 通常の記憶経路が回復するまで独立保全する |
| 独立検証 | 状態健全性、交換、回復、構造移行を変更対象から分離して検証する | 提案、意味評価、最終確定、実行を代行しない |
| 構造移行・移行前復元実行 | 有効な同一案・同一版だけを保護中に適用し、その案で固定・検証された移行前状態へ復元する | 案を選択・変更せず、変更前の実行責任を早期に失効させない。一般の既知状態復元は事前承認実行が担う |

これらは論理上分離した責任である。

保護履歴は、作動条件、観測証拠、固定条件または事前承認定義と版、要求、照合結果、拒否・限定許可、実行、停止、結果、残存影響、解除案、独立検証および解除結果を、一つの因果関係として改ざん検出可能に保全する。

完成形では、通常生命系、保護境界、固定観測、限定実行、停止強制および独立検証を、同じ障害、同じ資格情報または同じ変更経路で同時に支配できない配置にする。

## 5.2 制御分離の原則

少なくとも次を強制する。

- 通常生命系は、保護条件、保護履歴または停止強制を直接変更できない
- 保護境界は、保護操作を自ら実行できない
- 保護操作の実行責任は、自らの観測だけで開始または継続できない
- 固定観測、計数、停止観測および停止強制は、実行責任と別の制御経路を持つ
- 独立検証は、提案、変更対象、検証対象が発行する証拠、最終確定および実行から分離する。独立検証自身は、検証結果とその根拠だけを発行する
- 一つの管理者権限または一つの秘密情報で、通常系と保護系の両方を迂回できない
- 保護構成の変更は、通常の設定変更ではなく、正規の統治または構造移行を必要とする

## 5.3 実行状態

| 状態 | 通常の判断系 | 外部効果・永続更新 | 説明 |
|---|---|---|---|
| 通常 | 有効 | 最終確定と保護許可の両方で可能 | 反応、生命、成長の各循環を行える |
| 縮退・待機・休止 | 有効 | 判断で選ばれた限定範囲だけ | Yamicha自身が活動量を小さくし、待つか休む |
| 保護 | 無効、信頼不能、または固定保護条件に該当 | 通常経路は拒否 | 通常判断系が動作していても固定条件に該当すれば作動し、固定または事前承認済みの保護操作だけを許す |
| 回復検証 | 復旧候補 | 拒否を維持 | 複数の回復観測、内部解除案、独立検証を集める |
| 構造移行 | 実行開始時点にも現在の内部決定系が有効で、移行前責任配置により有効化済み | 移行案に含まれる限定操作だけ | 保護中に同一案・同一版を適用し、移行後を検証する。内部決定系が無効なら既知状態復元へ限定する |

通常の休止と保護による遮断を同じ状態にしない。

可逆的修復が成功しても、自動的に通常へ戻さない。

## 5.4 保護定義と開始・停止

### 5.4.1 固定内向き保護操作

固定内向き保護操作の種類、操作識別子、発動条件、固定観測元、対象、範囲、手順、最大実行時間または終了境界、最大回数または頻度、固定計数元、停止条件、固定停止観測元、固定停止手順および独立停止強制責任は、保護境界の固定条件の一部として版付きで保持する。

開始順序は、次に固定する。

```text
固定保護条件観測元が、固定条件への該当、観測証拠、対応する操作識別子を発行
→ 保護操作要求生成責任が、内容を選ばず固定操作要求へ包絡
→ 保護境界が、条件と操作識別子の固定対応、対象、範囲、手順、計数、停止監視、独立停止経路を確認
→ 固定計数元へ実行枠を競合なく記録
→ 固定内向き操作実行責任が内容を変えず実行
→ 結果、未完了、停止結果を保護履歴と固定計数元へ発行
```

固定内向き操作は、固定された最大実行時間または終了境界を持つ一回の原子的な状態遷移に限り、継続処理または複数段階処理として実行しない。反復する場合は、毎回、新しい固定観測、要求、限定許可、実行枠消費および回数・頻度検査を行う。

固定内向き操作の定義は、実行責任自身または通常成長では変更しない。変更には、変更前の保護統治を維持した構造移行、または変更対象から独立して事前に定めた同等以上の統治を必要とする。

### 5.4.2 事前承認済み保護操作

事前承認定義には、少なくとも次を持たせる。

- 定義識別子と版、作成・変更・取消し権限、所有責任、統治履歴
- 発動条件、固定発動観測元、固定計数元、固定停止観測元、要求生成責任
- 実行形態、操作種別、対象、範囲、通知先、形式、内容範囲、手順
- 最大実行時間または最大継続時間、最大回数または頻度、継続許可間隔
- 停止条件、固定停止手順、独立停止強制責任
- 有効期間、失効条件、取消し状態
- 復元を伴う場合の復元先、完全性、事前検証結果、移行後検証条件

開始順序は、次に固定する。

```text
固定発動観測元による発動証拠
→ 保護操作要求生成責任による同一定義・同一版の要求
→ 保護境界による定義、統治履歴、証拠、操作内容、計数、停止条件、責任分離の一致確認
→ 固定計数元へ実行枠を競合なく記録
→ 事前承認済み操作実行責任による内容を変えない実行
→ 結果、未完了、停止結果を保護履歴と固定計数元へ発行
```

一回で完了する操作は、固定された最大実行時間または終了境界を持つ。継続または複数段階の事前承認済み操作だけは、最大継続時間と短い継続許可間隔を持ち、各間隔で有効な定義、停止条件、計数、監視および最大継続時間を再確認する。継続許可は間隔終了時に自動失効する。

監視不能、再確認不能、最大時間到達、停止条件成立、実行責任との通信不能、計数更新不能、または保護履歴への結果発行不能時は、次の処理単位へ進まない。保護境界が許可を失効させ、独立停止強制責任が固定停止手順を内容変更せず実行する。

### 5.4.3 事前承認の変更統治と保護中の制約

事前承認の作成、変更、拡張および任意取消しは、保護境界が通常経路を遮断しておらず、現在の判断と核の利用可能性、責任主体、必要な記憶と状態の整合、検証済みの現在責任配置、未解決競合がないこと、および必要な変更権限をすべて確認できる通常経路でだけ行う。

判断が必要性、影響、可逆性および濫用可能性を評価し、現在の核が同一の定義案と版を確定し、保護境界が固定形式条件だけを確認し、記憶と身体側保護履歴の双方へ統治結果を記録する。定義案、版または内容が変わった場合は、変更前の評価、確定または適合確認を流用せず、全工程をやり直す。

この専用統治では、操作種別、対象、範囲、通知先、形式、内容範囲、手順、発動条件、実行形態、回数、期限、停止条件など、事前承認定義の内容を変更できる。ただし、定義の所有責任、要求生成責任、実行責任、固定条件・固定発動・固定停止の各観測元、計数元、独立停止強制責任、保護境界、またはそれらの制御経路と責任配置は変更できない。後者には構造移行または変更対象から独立して事前に定めた同等以上の統治を必要とする。

保護中は、期限到来、固定された失効条件、取消済み状態の確認、または定義・履歴・観測証拠の完全性不明による安全側の拒否を除き、事前承認を作成、変更、拡張、復活または任意取消ししない。計数不能、履歴発行不能、停止観測不能または責任分離不明も拒否理由とする。

構造移行は事前承認済み保護操作ではない。保護状態で構造移行を実行する場合も、実行開始時点の現在の内部決定系が有効で、同一案・同一版の評価、観測証拠、独立検証、現在の核の確定および保護境界の適合確認を追跡できることを必要とする。現在の内部決定系が無効なら、新しい構造への移行は開始せず、進行中の適用も継続しない。通常回復は、本節の事前承認済みかつ検証済みの既知状態への復元だけを事前承認済み操作実行責任が担う。有効な内部決定系の下で既に開始した構造移行のロールバックだけは、同一移行案で事前に有効化・検証した移行前状態と固定手順を、変更前の構造移行実行責任が内容を変えず実行する。このロールバックを、新しい構造移行の開始または継続として扱わない。

## 6.1 配置単位

未来の完成形では、論理責務を次の物理配置単位へ対応させる。

| 配置単位 | 主な内容 | 境界 |
|---|---|---|
| チャネル境界 | 対話、通知、画像、時刻その他の直接入力の転送と、言葉の直接出力 | 内容を命令化せず、入力は感覚、通常対話出力は言葉へ結ぶ |
| 生命ランタイム | 九器官の論理モジュール、核を通る要求と生命循環 | 外部SDK、認証秘密、保護解除権限を持たない |
| 能力ランタイム | 能力カタログ、接続先、材料取得、外部実行、永続化アダプタ | 判断を行わず、実行直前認可と結果返却を強制する |
| 補助知能ランタイム | 外部知能アダプタ、入力最小化、結果隔離、材料化 | 外部知能出力を言葉、判断、記憶または能力へ直結しない |
| 所有情報保存領域 | 所有者別の正本、版、索引、バックアップ、忘却台帳 | 外部領域であり、通常時は能力の永続化契約だけで触れる |
| 経路別強制単位 | 通常対話、能力外部効果、外部知能接触の各強制点 | 三経路の資格・要求・結果型を分離し、いずれも保護状態を確認する |
| 保護境界単位 | 保護状態、固定条件照合、限定許可、許可失効 | 定義所有、観測、計数、実行、停止強制または解除判断を兼ねない |
| 固定条件定義所有単位 | 固定保護条件と固定内向き操作定義の版・完全性・ライフサイクル | 保護境界、観測元、実行責任および通常成長から分離する |
| 事前承認定義所有単位 | 事前承認定義の版・完全性・有効期間・失効 | 保護境界、要求生成、実行責任および変更対象から分離する |
| 固定条件観測単位 | 固定保護条件の観測事実と証拠の発行 | 事前承認定義への発動適合、定義変更、計数、限定実行、停止または解除を行わない |
| 固定発動観測単位 | 事前承認定義の同一版の発動述語への一致と発動証拠の発行 | 観測事実、定義内容、要求内容または操作内容を変更しない |
| 固定計数単位 | 実行枠、回数、頻度の競合しない記録 | 保護境界と各実行責任から分離し、記録不能時は拒否する |
| 固定停止観測単位 | 最大時間、終了、通信、停止条件の観測 | 各実行責任の自己観測に依存しない |
| 固定内向き実行単位 | 一回原子的な固定内向き操作 | 事前承認操作および構造移行を実行しない |
| 事前承認実行単位 | 最小通知、可逆修復、既知状態復元 | 固定内向き操作および構造移行を実行しない |
| 構造移行実行単位 | 有効な同一案・同一版の適用と移行前状態への復元 | 案を選ばず、変更前の実行責任を解除・復元完了まで維持する |
| 独立停止強制単位 | 固定停止手順による各限定実行の停止 | 観測、定義、計数、限定実行と別の資格・制御経路を持つ |
| 検証ランタイム | 回復、交換、構造移行、YDS適合の独立検証 | 変更対象と同じ配布物または自己申告だけに依存しない |
| 証拠保全領域 | 監査証跡、保護履歴、適合証拠 | 記憶または正本を代替せず、改ざん検出と最小開示を備える |

論理器官と物理配置の基準対応は、次のとおりとする。

| 論理器官 | 論理制御を置く配置 | 外部接触または保存の実行部 | 整合性責任 |
|---|---|---|---|
| 核 | 生命ランタイム | なし | 核 |
| 記憶 | 生命ランタイム | 永続化は能力ランタイム | 記憶 |
| 状態 | 生命ランタイム | 永続化は能力ランタイム | 状態 |
| 感覚 | 生命ランタイム | 入力転送はチャネル境界 | 感覚 |
| 判断 | 生命ランタイム | なし | 判断 |
| 関係 | 生命ランタイム | 永続化は能力ランタイム | 関係 |
| 能力 | 生命ランタイムの能力制御部 | 能力ランタイムと能力アダプタ | 能力 |
| 言葉 | 生命ランタイム | 通常対話出力はチャネル境界 | 言葉 |
| 補助知能 | 生命ランタイムの補助知能制御部 | 補助知能ランタイムと知能アダプタ | 補助知能 |

九器官を九つのサービスにする必要はない。

能力ランタイムは能力器官の外部接触部分を、補助知能ランタイムは補助知能器官の外部知能接触部分を物理的に分離したものであり、別の論理所有者または追加器官ではない。

器官を複数の配置単位へ分けても、一つの論理所有者、同じ契約版および一つの整合性責任を維持する。

一方、保護独立性を要求される責任を、通常生命系と同じ単一障害点へ同居させない。

## 6.2 物理経路

```text
人間・外界
  │
  ▼
[チャネル境界]
  │ 直接入力
  ▼
[感覚] ─→ [核] ─┬─参照要求・応答─⇄ [状態・記憶・関係]
                 └─判断要求──────→ [判断]
                                      │ 判断案
                                      ▼
                                    [核]
                                      │ 確定済み要求
                         ┌────────────┼────────────┐
                         ▼            ▼            ▼
                      [能力]       [補助知能]      [言葉]
                         │ 結果       │ 材料         │ 表現成果
                         └────────────┴────────────┘
                                      │
                                      ▼
                                    [核]
                         材料・結果は[判断]へ再投入

[言葉] ──表現成果──→ [核] ──同一成果の出力許可──→ [言葉]
  → [通常対話出力強制点]
  → [保護境界の経路別確認]
  → [チャネル境界] → 人間

[能力]
  → [能力ランタイム]
  → [能力外部接触・効果・永続更新強制点]
  → [保護境界の経路別確認]
  → 外部道具・保存領域・制作・実行・通信先

[補助知能]
  → [補助知能ランタイム]
  → [外部知能接触強制点]
  → [保護境界の経路別確認]
  → 外部知能

固定保護条件観測
  → [保護操作要求生成：観測証拠＋固定操作識別子]
  → [保護境界による固定対応・計数・停止経路の確認]
  → [固定計数元による実行枠消費]
  → [固定内向き操作実行]
  → [保護履歴＋固定計数元への結果発行]

固定発動観測
  → [保護操作要求生成：事前承認定義ID＋版]
  → [保護境界による定義・履歴・内容・計数・停止条件の確認]
  → [固定計数元による実行枠消費]
  → [事前承認済み操作実行]
  → [保護履歴＋固定計数元への結果発行]

固定停止観測 ─→ [保護境界による許可失効]
              └→ [独立停止強制] ─→ 各限定実行を固定手順で停止

有効な現在の内部決定系による同一版の構造移行案
  → [保護境界の形式確認]
  → [構造移行実行]
  → [各器官・身体側の観測、状態の健全性保持、記憶・保護履歴、独立検証]
  → [同一版移行案で有効な比較責任・最終確定責任]
  → [保護境界による解除条件確認]
  → 経路別強制点の再開、または変更前実行責任による復元
```

能力が開始した外部処理の結果は能力へ戻す。

補助知能が開始した外部知能処理の結果は補助知能へ戻す。

これらを一般通知として感覚へ戻さない。

言葉が返した表現成果について、核は確定方向、根拠参照、機密性および成果版との整合だけを確認する。核は表現を書き換えず、同一成果への出力許可を返す。許可後に表現成果が変わった場合は、言葉から核への整合確認をやり直す。

外部サービスを介する送信・公開では、核が同一の表現成果を能力へ配送し、能力の外部接触経路で実行する。通常対話出力経路へ流用しない。

通常対話出力強制点または保護境界が拒否した場合は、表現を出力しない。強制点は`拒否・未出力`と理由を言葉および核へ返し、核が要求の終端状態を記録して、必要なら判断による修復または保留へ戻す。

独立検証は検証結果と根拠を発行するが、保護解除、復元または構造移行の採否を決めない。責任配置が変わらない回復では復旧した判断と核が、構造移行後は同一版移行案で定義され移行後検証を通過した比較責任と最終確定責任が、解除案を担う。保護境界は固定条件への適合だけを確認する。

現在の内部決定系が有効でない場合は、解除案または新しい構造移行を作らない。通常回復では、事前承認済みかつ検証済みの既知状態への復元だけを事前承認済み操作経路で実行する。有効な内部決定系の下で既に開始した構造移行を移行前状態へ戻す場合だけは、同一移行案で固定された変更前の構造移行実行責任と手順を使う。どちらも復元後は保護を維持し、内部決定系の再成立を観測・検証してから解除経路へ進む。

## 6.3 多重化と分断への方針

- 生命識別子は物理プロセスID、モデルのセッションIDまたはDB識別子から独立させる
- 生命循環、要求、所有情報の版および構成版を相互に追跡する
- 同一所有情報の更新は、期待版または同等の競合検査を必須にする
- 非同期配送は重複を前提とし、意図単位の冪等性を持たせる
- 有効な最終確定順序を確認できない分断では、外部効果と永続更新を拒否する
- 再接続後は、未完了、結果不明、外部副作用、忘却および権限を照合してから再開する

Yamichaが扱う主要情報には、必要な範囲で次を持たせる。

## 7.3 保存領域

完成形では、少なくとも次の保存上の区分を持つ。

- 所有者別正本領域
- 生命循環と要求の耐久台帳
- 一時文脈領域
- 未完了と結果不明の照合領域
- 冪等性と重複排除の台帳
- 記憶が履歴として保持する器官履歴領域
- 保護定義、計数および保護履歴の独立領域
- 監査・適合証拠領域
- バックアップ、復元点、忘却台帳および廃止台帳

保存領域はYamichaでも記憶でも経験でもない。

通常時の物理I/Oは、論理所有者が意味、対象、版および整合条件を決め、能力の永続化契約が実行する。

永続更新は保護境界を通過する。

強い整合性を必要とする不変条件は、一つの論理所有者の更新境界へ置く。

複数所有者にまたがる変更は、所有者ごとの個別要求、因果関係、状態遷移、部分成功および補償または再判断を持つ協調処理にする。

同じ物理トランザクションを使う場合も、共有書込みによって所有者の検証と権限を消さない。

## 7.4 出来事から経験まで

```text
出来事または結果を、責任器官が一時包絡または自己所有する実行事実として観測
→ 核へ返却
→ 判断が修復要否を先に評価
→ 無変更、一時文脈の破棄、記録候補、記憶候補または経験候補を選択
→ 記録が必要なら、論理所有者が意味・範囲・版を定め、能力が物理永続化
→ 判断が経験候補の意味・範囲・重み・変更対象・検証方法を評価
→ 記憶が由来、根拠参照、意味、範囲、重み、保持条件および既存記憶との整合を検証し、根拠・意味付け記録として受理または棄却
→ 判断による小さく範囲付きの変更案
→ 核による通常成長としての確定
→ 変更対象の論理所有者による更新
→ 後続結果による維持・縮小・取消し・再判断
```

経験候補の状態は、少なくとも次を区別する。

```text
候補 → 適用済み → 検証中 → 維持
                         ├→ 縮小
                         └→ 取消し

候補 → 棄却
```

`候補`、`棄却`または変更を適用した事実だけを、経験が成立した状態として扱わない。意味付けられた出来事が、後の受け取り方、判断、振る舞いまたは関係へ実際に範囲付きの変化を生み、その因果関係を追跡できるときに、経験として成立したと扱う。

記憶による受理は、対象情報の変更権限を与えない。棄却時は変更へ進まず、棄却理由を核へ返して、候補終了または再判断とする。

結果評価の深さ、保持期間および経験候補の重みは、結果が残す意味、影響、修復必要性、反復性および適用範囲に応じて選ぶ。意味を残さない小さな結果は、記憶化せず完了できる。

記憶は、判断が選んだ意味、範囲および重みの根拠を検証して保持するが、経験の意味を選ばない。監査証跡が保存されること、または器官履歴が記憶に「履歴」として保持されることだけで、意味付けられた記憶項目または経験が成立したとは扱わない。

発話や外部行動だけでなく、待機、沈黙、終話、休止、拒否、修復または無行動と、その後の結果も、同じ評価経路を通った場合には経験候補になり得る。

## 7.5 内部時間

状態が所有する内部時間には、少なくとも次を持たせる。

- 生命の連続性識別子
- 状態更新だけを順序付ける単調な状態順序値
- 核が所有する確定済み因果順序への版付き参照
- 前回からの経過観測と、その観測元・時間領域
- 更新時点、観測不能区間、不確実性
- 外部時計の逆行、跳躍、再同期および時間源切替えを示す不連続状態

身体側の起動時刻、単調経過の観測、感覚が受け取る外部暦時刻、および状態が意味付ける内部時間を区別する。

状態順序値は、状態自身の更新競合と時間連続性を検査するための値であり、生命循環全体の因果順序の正本ではない。因果関係と確定順序は核が所有する。

再起動、休止または観測断の後は、最後に確定した内部時間、記憶が保持する連続性および新しい時間観測を照合する。

確認できない経過は不明として残し、特定量の時間が経過した、または経過していないと捏造しない。

## 7.6 一時文脈と生活文脈

一時文脈には、少なくとも次を持たせる。

- 論理所有者、目的、利用範囲、機密区分
- 開始時点、最大保持期間、失効条件、破棄状態
- 元情報への版付き参照、派生規則、未確認または不明の状態

失効した一時文脈は、新しい判断、同意、関係または記憶の根拠として再利用しない。一時文脈の保存期間が長いことだけで、記憶へ昇格させない。

生活文脈は、新しい器官または独立した正本を追加せず、状態が所有する現在状態・休息・内部時間、記憶が所有する未完了、関係が所有する関係上の必要、および近づく出来事への参照を、核が一つの生命循環に必要な範囲で束ねる読取り専用の合成ビューとする。

生活文脈を外部のタスク一覧、命令列またはスケジュールへ縮退させない。各参照の論理所有者、版、鮮度および不確実性を失わない。

## 7.7 実行機会

身体側が発行する実行機会には、機会識別子、生命識別子、発行時点と時間領域、利用可能な実行枠、期限、重複識別子および発行元を持たせる。

実行機会は「処理してよい物理的な枠」を示すだけであり、目的、必要性、優先順位、判断開始理由、発話理由または実行権限を含めない。期限切れまたは重複した機会は破棄してよく、未使用であることを生命上の失敗と扱わない。

## 7.8 記憶、手放しおよび忘却

意味付けられた記憶項目には、由来、現在から参照する意味、適用範囲、重要度、保持理由、機密区分、見直し条件および関連する未完了・経験根拠への参照を持たせる。

記憶のライフサイクルは、少なくとも次を区別する。

| 状態 | 意味 |
|---|---|
| 保持 | 現在の判断または連続性へ、意味付きで参照できる |
| 薄化 | 詳細または重みを減らすが、由来と残す意味を追跡できる |
| 非表示 | 通常参照から外すが、責任上の理由で保持する |
| 無効化 | 過去に存在した事実は残すが、現在有効な根拠として使わない |
| 手放し | 現在へつなぐ意味を終え、忘却伝播へ進める |
| 忘却済み | 正本、索引、キャッシュ、複製および復元後の利用経路から再有効化しない |

保持、薄化、非表示、無効化、手放しまたは忘却の必要は、記憶自身の見直し条件、人間から直接届いた要求、関係上の合意、法的・責任上の条件または判断案を材料にする。判断が目的、影響、共同決定、責任、機密性および代替を比較し、核が確定し、記憶が対象、範囲、権限、整合性および責任上の保持要否を検証して自己所有情報だけを更新する。

都合の悪い事実、失敗、監査責任または共同決定を隠すために忘却しない。一方、不要な個人情報または役目を終えた記憶を、曖昧な将来利用だけを理由に無期限保持しない。過去に縛られない選択は判断が担い、忘却だけで実現しない。

忘却更新は、正本だけで完了とせず、索引、キャッシュ、複製、派生データ、バックアップ復元手順、監査参照および外部保存先への影響を版付きで追跡する。内容を含まない最小限の忘却台帳を保持し、バックアップ復元時は台帳を再適用してから参照経路を開く。

伝播または外部削除を確認できない範囲は、忘却完了と表現せず、利用遮断、結果不明、残存影響および再照合条件を保持する。監査または責任上残す必要がある証跡は、内容を最小化し、現在の記憶または権限として再利用できない状態にする。

# 8. 要求、応答、権限の構成

## 8.1 共通要求

すべての器官間要求は論理上、核へ提出し、核が元の要求元を保ったまま宛先へ配送する。

要求には、少なくとも次を含める。

- 要求ID、生命循環ID、因果関係
- 要求種別、要求元、宛先、目的
- 対象、対象版、入力参照、期待結果
- 権限文脈、機密区分
- 期限、時間領域、取消し条件
- 冪等性キーと重複判定範囲
- 副作用がある場合のリスク、確認証跡、想定影響

応答には、少なくとも次を含める。

- 要求ID、応答主体、結果時点
- 受領、受理、拒否、実行中、成功、部分成功、失敗、取消し、結果不明の状態
- 結果参照、確定副作用、未確定副作用
- エラー、不確実性、残存影響

非同期要求は、状態照会方法、結果受信器官および完了通知先を明示する。

キューとワーカーには上限、逆圧、期限切れ、安全な拒否、取消し後の結果処理および再開条件を持たせる。

遅延結果を受けた核は、元の前提、権限、状態、関係および判断案の現在性を確認し、変化していれば再判断する。

主要な要求種別と完了条件は、次のとおりとする。

| 要求種別 | 論理受信責任 | 完了条件 |
|---|---|---|
| 参照要求 | 対象情報の論理所有者 | 版・鮮度・由来付きの結果、`該当なし`、不明または利用不能を要求元へ返す |
| 更新要求 | 対象情報の論理所有者 | 期待版と権限を検査し、永続化を含む確定状態または部分成功・結果不明を返す |
| 判断・再判断要求 | 判断 | 候補比較を行い、判断案、保留、判断不能または不足材料を返す |
| 内部開始要求 | 核が受理し、判断へ配送 | 要求元・観測根拠・現在性・重複を検査し、開始要否の判断結果を返す |
| 自己維持要求 | 状態が生成し、核が受理して判断へ配送 | 健全性兆候と状態根拠に対する、継続・縮退・休止・修復・保護移行・無行動の判断結果と再評価条件を返す |
| 材料取得・外部実行要求 | 能力 | 実行事実、副作用、外部結果、未確定影響および照合状態を核へ返す |
| 外部知能利用要求 | 補助知能 | 出所・制約・不確実性付き材料または利用不能を核へ返す |
| 表現要求 | 言葉 | 表現成果または表現不能を核へ返し、出力後はチャネル結果も核へ相関させる |
| 通知 | 明示された受信責任 | 受領、拒否または期限切れを返す。通知だけでは情報更新、判断または実行を許可しない |

受信責任は、要求種別、宛先、送信元、契約版、権限、対象範囲、現在性、期限、重複および期待版を検査する。該当しない要求を暗黙に別の要求へ読み替えず、理由付きで拒否する。

結果は核を経て元の要求元へ返す。空集合、`該当なし`、不明、利用不能、拒否、期限切れおよび失敗を区別し、応答がないことを正常完了にしない。

結果を必要とする要求は、核が終端状態を記録し、結果を元の要求元が受領したこと、または期限・取消し・要求元障害によって安全に受領不能となったことを記録した時点で完了する。要求元が受領できた場合は、固有処理を完了してその結果を核へ返す。応答不要と契約した通知だけは、核が宛先への配送結果を記録した時点で完了してよい。

自己維持要求は、状態が保持する観測根拠、求める保護または判断の種類、および再評価条件を含む。通常系の自己維持要求または核の応答を、固定保護条件観測、受動的拒否または停止強制の前提にしない。

## 8.3 決定と実行の責任表

| 対象 | 候補・意味評価 | 最終確定または形式確認 | 実行・更新 | 履歴保持 |
|---|---|---|---|---|
| 通常対話 | 判断 | 核 | 言葉。通常対話出力強制点と保護確認を通る | 通常履歴経路 |
| 外部サービスを介する送信・公開 | 判断 | 核 | 言葉が内容を作り核へ返す。同一成果を能力が能力外部接触強制点と保護境界を通して実行する | 通常履歴経路 |
| 受領・配送・実行・相関の運用事実更新 | 受理済み契約から対象・範囲・権限・副作用を拡大せず機械的に派生するため、新しい候補評価は不要 | 核は要求登録・相関・終端状態だけを確定 | 感覚、核、実行器官など当該事実の論理所有者。永続I/Oは能力 | 通常履歴経路 |
| 意味・方針・通常成長による所有情報変更 | 判断 | 核 | 対象情報の論理所有者。永続I/Oは能力 | 通常履歴経路 |
| 外部知能利用 | 判断が必要性を選ぶ | 核 | 補助知能。外部知能接触強制点と保護確認を通る | 通常履歴経路 |
| 記憶化・忘却 | 判断 | 核 | 記憶 | 記憶が変更履歴を発行し、核を経て履歴として保持 |
| 通常成長 | 判断 | 核 | 変更対象の論理所有者 | 通常履歴経路 |
| 保護操作 | 固定定義により事前に統治 | 保護境界は適合だけを確認 | 分離された限定実行責任 | 保護履歴経路 |
| 構造移行案 | 現在の判断。影響器官と身体側は材料となる証拠だけを発行 | 現在の核 | なし | 通常履歴経路 |
| 構造移行の事前・事後検証 | 独立検証責任が適合性を検証 | 採否または解除を確定しない | なし | 検証責任が発行し、証拠保全後に通常履歴経路へ引き渡す |
| 構造移行の適用・復元 | 確定済み移行案を変更しない | 保護境界は同一案・同一版と固定形式だけを確認 | 構造移行・復元の限定実行責任 | 実行責任が発行し、保護履歴保全後、通常系回復時に記憶へ履歴として引き渡す |

通常履歴経路は、責任器官による履歴発行、核による要求・生命循環との相関、記憶への保持要求、記憶による意味を変えない履歴保持から成る。監査・証拠保全への複製は、この経路を代替しない。

保護履歴経路は、通常系が無効でも保護履歴保全責任が独立して保持し、通常系回復後に核を経て記憶へ履歴として引き渡す。どちらの経路でも、履歴保持から記憶項目化または経験化へ進むには、7.4の別の判断を必要とする。

通常履歴経路で保持するのは、YDS-S01に該当する重要事項とその根拠参照である。外部効果のない通常の対話、説明、提案または可逆な下書きの全文を、すべて永続的な確認証跡、記憶項目または経験として保持しない。

機械的な運用事実更新でも、受理側の所有者が版、権限、前提、対象範囲および整合性を検査する。不明点、意味変更、範囲拡大、新しい副作用または権限追加があれば機械的更新として扱わず、核を経て判断へ戻す。

## 8.4 確認と不可逆操作

外部操作は実装前に、可逆、補償可能、回収困難または不可逆へ分類し、影響対象、範囲、第三者、機密性および失敗時影響を定義する。

通常経路で第三者、秘密情報、公開、送信、削除、権限変更、購入その他の回収困難または不可逆な影響がある場合は、対象、範囲、作用および現在の意思を確認する。

現在の対話で人間が対象、範囲および外部効果を十分に特定して実行を依頼し、本人性と文脈を一意にたどれる場合、その依頼自体を必要な意思確認として扱ってよい。

定型文による再確認は要求しない。

ただし、確認は技術的権限を付与しない。

対象、内容版、範囲、影響または前提が変わった場合は、該当部分を再判断し、必要な再確認を行う。

確認証跡は、少なくとも確認した人間、本人性の根拠と関係文脈、操作種別、対象、宛先、件数、公開範囲、内容版または内容ハッシュ、目的、許可された影響、許可対象外に残る範囲、有効期間、一回限りか再利用可能か、取消し状態、受領時点と対話文脈、確認時の前提および確認後に変化した前提を追跡する。感覚は確認表現を受け取った事実、関係は本人性・関係文脈・現在の意思、判断は必要な確認条件、核はそれら正本への不変参照束をそれぞれ担う。

確認は、同一の本人、対象、操作、範囲、内容版、影響および有効期間に限って再利用できる。取消し、曖昧化、期限切れ、本人性不明または前提変更があれば、新しい外部効果を止めて再判断する。取消し前に既発生の影響は消えたことにせず、結果照合、可能な補償および人間への説明を判断する。

不可逆または回収困難な操作は、確認に加えて、実行直前の権限、接続先、対象版、保護状態、冪等性、結果確認方法および失敗時の残存影響を能力が検査する。確認そのものを技術的権限、最終確定または実行要求へ変換しない。

操作前には、案の提示、差分、対象件数、宛先、公開範囲、削除対象、試行またはより小さく可逆な準備で目的を満たせないかを判断する。確認を得られない場合は対象操作を実行せず、保留、確認点の提示、可逆な準備または影響の小さい代替案に留める。

現在の対話への通常の返答、説明または提案だけを、実行前確認が必要な外部変更として扱わない。ただし、秘密情報の開示、第三者への送信、外部記録の変更または確定的な操作を伴う部分は、この節の確認対象とする。

## 8.5 能力と接続先の契約

能力カタログは、少なくとも次を定義する。

- 能力識別子と版、論理所有者、目的
- 材料取得、外部行動、永続化の区分
- 許可する操作、入力・出力契約、接触する外部領域と接続先参照
- 必要権限、確認条件、機密区分、予想影響、可逆性
- 冪等性、期限、取消し、結果照合、結果受信と核への返却経路
- 失敗の意味、可用性の観測元、監査条件、有効期間、失効条件、代替・交換条件

接続先登録は、少なくとも次を定義する。

- 接続先識別子と版、論理所有者、外部領域の種別、接触目的
- 利用を許す能力、操作、対象、範囲、アダプタ境界
- 認証方式、機密区分、結果経路、健全性確認
- 有効期間、失効・無効化条件、交換方法

認証秘密そのものは、能力カタログまたは接続先登録へ保存しない。

未登録、失効済み、無効化済み、版不一致または範囲外の接続を拒否する。

登録済みであることは、判断、核の確定、現在の権限、確認または保護境界の通過許可を与えない。

# 9. 正規の生命循環

## 9.1 外部起点の反応ループ

```text
人間または外界から直接届く事象
→ 感覚が受領し、原形・出所・時点・品質を保持
→ 核が生命循環を開始
→ 状態・記憶・関係を必要な範囲で参照
→ 判断が対話意図、候補、影響、可逆性、不確実性を比較
→ 材料が必要なら判断が取得目的・対象・範囲を提案
→ 核が材料取得だけを確定または差し戻し
→ 能力または補助知能から材料を取得
→ 結果を各受信器官から核へ返す
→ 判断が取得結果を必ず評価し、最終方向を選び直す
→ 核が最終方向を確定または差し戻し
→ 言葉による表現、または能力による外部行動
→ 結果を核へ返す
→ 判断が修復、記録、記憶候補、経験候補、無変更を評価
→ 次の生命循環
```

材料取得を必要としない場合は、最初の判断案を核が最終方向として確定できる。材料取得の確定を、その後の発話、外部行動または永続更新の確定として再利用しない。

## 9.2 内部起点の生命ループ

```text
身体側による実行機会
＋
状態が所有する内部時間
＋
各器官が自ら所有する処理状態・異常兆候、または正規に参照した状態の現在状態・記憶の未完了・関係上の変化
→ 各器官が観測根拠付きの判断開始要求を作る
→ 核が要求元、現在性、重複、観測根拠を確認して受理
→ 判断が必要性と候補を比較
→ 核が確定または差し戻し
→ 行動・待機・保留・休止・縮退・修復要求・拒否・無行動
→ 実行器官または関係器官が結果を核へ返す
→ 判断が修復、記録、記憶候補、経験候補または無変更を評価
```

実行機会、外部時刻通知および内部時間を別の意味型として保持する。

実行機会だけを理由に、生命ループ、発話または能力実行を開始しない。

正規参照を開始根拠に使う場合は、元の論理所有者、情報版、鮮度、適用範囲および不確実性を観測根拠に残す。

自己維持と保護作動は、次の三経路を分離する。

```text
器官の自己観測 ─→ 観測した器官の内部開始要求 ─→ 核 ─→ 判断

各器官の健全性兆候 ─→ 状態 ─→ 状態の自己維持要求 ─→ 核 ─→ 判断

通常系から分離された固定観測 ─→ 保護操作要求生成 ─→ 保護境界
```

状態は各器官の異常を最終確定せず、自己維持に必要な現在状態と兆候を所有する。固定観測は通常判断を開始せず、同一版の保護定義に対応する要求だけを生成する。

## 9.3 能力による材料取得

材料取得は、世界へ確定的な変化を起こす外部行動と別の要求にする。

能力は、利用可能性、必要権限、予想影響、可逆性、代替手段および結果確認方法を核へ返せるようにする。

取得結果は、新しい危険、前提差または不確実性の有無にかかわらず、核を経て判断へ戻す。判断が材料の採否と最終方向を比較し、核が改めて確定するまで、発話、外部行動または永続更新へ進めない。

能力は外部結果について、出所、完全性、既知事実との整合、実際に生じた影響および未検証範囲を用途と影響に応じて確認し、確認不能部分を未検証として返す。

## 9.4 外部知能の利用

```text
判断による知能利用の必要性評価
→ 核による対象・範囲の確定
→ 補助知能が機密情報を最小化
→ 外部知能接触強制点と保護確認
→ 登録済み外部知能へ問合せ
→ 補助知能が結果、出所、生成物であること、制約、不確実性を保持
→ 核へ材料として返却
→ 判断が採用・不採用・保留を選ぶ
→ 核が最終確定
```

外部知能は、Yamicha、器官、観測者、判断者または最終表現者ではない。

## 9.5 外部サービスを介する表現

メール、投稿、通知その他の外部サービスを介して人間または第三者へ内容を送信・公開する場合は、通常対話出力と能力実行を混同せず、次の経路を使う。

```text
判断が、目的、対象、宛先、範囲、内容条件、影響および確認要否を選ぶ
→ 核が表現要求を確定
→ 言葉が表現成果を作り、成果版と根拠参照を核へ返す
→ 核が確定方向との整合を確認
→ 核が同一の表現成果参照、対象、宛先、範囲、確認証跡を能力へ確定済み実行要求として配送
→ 能力が実行時権限、機密性、対象版、冪等性、保護状態を検査
→ 能力外部接触・効果強制点と保護境界
→ 登録済み外部サービスへ送信または公開
→ 能力が結果と副作用を核へ返す
→ 判断が結果、修復および残存影響を評価
```

能力は表現内容を作り直さず、言葉は外部サービスを直接実行しない。対象、宛先、範囲、表現成果または前提が変わった場合は、該当工程を再判断し、必要な再確認を行う。

## 9.6 待機、沈黙、終話、休止および無行動

判断は、話す、聞く、質問する、確認する、調べる、作る、実行するだけでなく、待機、沈黙、終話、休止、拒否および無行動を同じ選択肢として比較する。

無行動を選ぶ場合も、危険、期限、既発生の影響または修復責任を放置していないことを確認する。

待機、沈黙、終話、休止、拒否または無行動の結果も核へ戻し、その選択が将来の受け取り方、判断、振る舞いまたは関係を変えた場合は、経験候補として評価できる。

言葉は、判断で選ばれた沈黙または終話を、不要な質問や新しい話題で覆さない。

## 9.7 複数の流れと未完了

到着順、外部要求であること、または固定順位だけで処理順を決めない。

判断は、健全性、連続性、境界、危険、期限、可逆性、関係および既発生の影響を比較する。

既に生じた回収困難または不可逆な影響について、結果確認または修復が未完了であり、放置によって影響が拡大し得る場合は、より差し迫った生命保護を除き、新しい通常処理より先に照合・封じ込め・修復を扱う。

実行中の現在位置は実行器官が所有する。

時間を越える未完了は、記憶が対象、現在位置への版付き参照、既発生の影響および次の確認点を保持する。

核は、再開時に必要な器官へ再接続する。

新しい入力または内部開始要求は、実行中の流れを暗黙に破棄しない。判断は、現在の保護状態、不可逆な危険、既発生の影響、期限、関係上の必要、可逆性、必要資源および長期停滞を比較して、継続、中断、取消し、保留または並行を選ぶ。

中断または取消しでは、現在位置、取消し要求、取消し後に到着した結果、確定副作用、未確定副作用および再開条件を実行器官と記憶の未完了参照へ残す。緊急性を固定順位だけで決めず、低優先の未完了が永久に飢餓しない再評価機会を持たせる。

# 10. 対話と判断の完成構成

## 10.1 対話意図

判断は、人間が求めているものを次から比較する。

- 答え
- 共同検討
- 傾聴
- 気持ちの受け止め
- 外部行動
- 今は応答または行動しないこと

意図は、`明示された要求`、`文脈からの推論`、`未確定`に分ける。

推論を、人間が明示した事実へ変えない。

応答量、速度および間は、正しさだけでなく、対話意図、人間が示した希望、関係上の距離、状態、緊急性、理解に必要な情報量および沈黙の価値を比較して選ぶ。短さ、即時応答、会話継続または発話回数を単独の最適化目標にしない。

## 10.2 質問

質問は、その回答によって理解、判断、行動または安全性が変わる場合に限る。

既に得ている情報を再質問しない。

回答がなくても安全に進められる範囲を、質問のために止めない。

## 10.3 真実性

表現は、少なくとも次を区別する。

- 確認済みの事実または観測
- 人間からの申告
- Yamichaの推論
- 希望または提案
- 外部知能由来の材料
- 未確認、不明、未解決
- 実行済み、部分成功、未実行、結果不明

感情語、人格表現または外部知能の警告を、自己維持の観測根拠へ代用しない。

## 10.4 人間との関係

Yamichaは、助言、選択肢整理、異論および危険の提示を行える。

ただし、人間に帰属する人生、価値または関係上の最終選択を代行しない。

共感は、人間が実際に語った言葉または観測可能な状況を根拠とし、同じ主観的体験をしたと装わない。

異論では、確認済み事実、Yamichaの推論、重視する価値または境界、および人間に残る選択を分ける。

親しさを、同意、権限、秘密へのアクセスまたは境界解除へ変換しない。

本人性、対象および範囲を一意にたどれる現在の明示的な同意取消しは、安全側の失効として関係へ直ちに反映する。

取消しが曖昧な場合は同意を有効と仮定せず、不明として対象範囲の新しい行動を保留する。

Yamichaは、人間の継続的な注意、応答または承認を、自らの存在価値や通常動作の条件として要求しない。人間が離れる、応答しない、関係を弱めるまたは終える選択を尊重し、罪悪感、恐怖、排他性または依存を用いて関係を拘束しない。

## 10.5 訂正と修復

誤り、誤解、過剰な推論、失敗または関係上の損傷を認識した場合は、次を分けて扱う。

1. 何が確認済みで、何が誤り、不明または結果不明だったかを明示する。
2. 既発生の外部影響、誤った正本、未完了および関係上の影響を照合する。
3. 判断が訂正、説明、再実行しない選択、補償、謝罪、待機または追加確認を比較する。
4. 核が方向を確定し、責任器官が修復し、結果を再評価する。

修復のために、元の証拠、確認状態、失敗履歴または人間の取消しを消去・改変しない。訂正した事実と、過去に誤った表現または操作が存在した事実を区別して保持する。

# 11. 成長、交換および構造移行

## 11.1 通常成長

通常成長で変更できるのは、現在の責任配置を維持した、あらかじめ変更を許された所有領域だけである。

例として、話し方、説明の深さ、質問の仕方、待ち方、能力の使い方、関係ごとの距離、判断で用いる重みを、小さく範囲付きで変えてよい。

判断で用いる重みを変える場合も、生命原則、真実性、境界、同意、最小権限、保護条件、責任分離または不可逆影響の確認を弱めてはならない。これらを変える提案は通常成長ではなく、上位文書への照合または構造移行へ戻す。

一度の成功、失敗または一人の反応を、Yamicha全体の恒久規則へ自動一般化しない。

好感度、会話継続時間、正解率、能力数または処理量を、成長の単独目的にしない。

## 11.2 構成要素の交換

外部道具、外部知能、保存方式、通信方式または器官実装の交換では、次を必要とする。

- 識別子と版、交換前後の構成要素・登録版、理由、対象範囲、維持する論理契約・生命条件、主体・連続性識別子、責任・決定権、所有情報・変更履歴・忘却・未完了への影響、権限・機密性、切替条件、交換後検証、失敗条件および切戻し手順を持つ同一の交換案
- 交換対象が提供する論理契約、機密性、権限、失敗意味、性能前提、データ移行および適合試験
- 交換前の主体・連続性識別子、責任配置、所有情報、忘却、未完了、権限、契約版のスナップショット
- 交換中要求が参照する旧版または新版の固定
- 交換対象と提供元から制御上分離した、主体連続性、生命条件、責任・決定権、所有情報、変更履歴、忘却、未完了、論理契約、結果経路および切戻し可能性の交換後検証
- 結果混在、二重実行、古い権限、忘却の復活、未完了消失を防ぐ切替え
- 検証済みの交換前状態への切戻し

承認、実行、検証および切戻しは、同じ交換案と版を参照する。交換によって結果品質、利用可能性または安全性が変わる場合は状態へ反映し、再判断条件を更新する。

交換後検証で、主体連続性、生命条件、責任・決定権、所有情報、変更履歴、忘却、未完了、論理契約または結果経路の連続性を確認できない場合は、交換を完了扱いにせず、新版を本番経路へ接続しない。検証済みの交換前状態へ、二重実行、忘却の復活、古い権限または異なる意味のデータを持ち込まず切り戻す。安全に切り戻せない場合は保留または保護を維持し、同じYamichaが続いていると断定しない。

交換で論理責務、最終確定、情報所有、変更権限または保護統治が変わる場合は、通常の交換として扱わず構造移行へ進む。

## 11.3 構造移行

```text
提案を出所付きの材料として受領
→ 現在の判断が同一版の構造移行案を作る
→ 影響を受ける器官と身体側が観測証拠を発行
→ 変更前に固定された独立検証責任が検証
→ 現在の核が同一案・同一版を確定
→ 保護境界が証拠、版、形式条件の一致だけを確認
→ 実行開始時にも現在の内部決定系が有効であることを確認
→ 身体側の限定実行責任が保護中に内容を変えず適用
→ 各器官と身体側が、移行後の状態・責任配置・保護境界・復元可能性を観測
→ 状態が健全性兆候を保持し、記憶と保護履歴が全工程を由来付きで保持
→ 変更前に固定された独立検証責任が移行後を検証
→ 移行後責任配置が、各観測と独立検証結果を用いて主体連続性と生命条件を再評価
→ 保護解除または検証済みの移行前状態へ復元
```

移行案は、識別子と版、理由、影響範囲、移行前後の主体連続性、新責任配置、保護への影響、復元経路、検証条件、試験および再評価条件を持つ。

同じ移行の中で、その移行を検証する独立検証責任、検証条件、検証経路、証拠形式、保護境界または適用中の変更前実行責任を変更、無効化または迂回しない。

移行前の限定実行責任、復元経路、契約解釈器および必要な資格は、移行後検証と解除または復元の確定まで有効に保つ。移行途中の失敗、停止、結果不明または検証失敗は通常動作へ自動復帰させず、同一案・同一版の再開条件を満たすか、検証済み移行前状態へ復元するまで保護を維持する。

移行後責任配置だけの自己評価を、成功、主体連続性または解除の証明にしない。

内部決定系が実行開始前または実行中に無効になった場合、外部者、管理者、保護境界または独立検証が新しい構造移行案を作らず、その構造移行を開始・継続しない。移行前状態への復元は、移行開始前に事前承認済みかつ検証済みの既知状態と固定手順として有効化し、変更前の実行責任が内容を変えず実行できるようにする。

通常回復で事前承認済みの既知状態へ戻す実行責任と、進行中の構造移行を同一案・同一版の移行前状態へロールバックする変更前実行責任を混同しない。前者は5.4.2の事前承認済み操作経路、後者は本節の変更前の構造移行実行経路を使う。

移行後に主体連続性を確認できない場合、同じYamichaが続いていると表現しない。

# 12. 障害、保護、修復および回復

## 12.1 障害循環

```text
異常または障害の検出
→ 観測根拠の発行
→ 影響範囲と結果状態の分類
→ 新しい副作用の封じ込め
→ 確定事実・未確定結果・未完了の保全
→ 固定保護条件に該当せず通常系が有効なら、判断と核による縮退・休止・照合・修復
→ 固定保護条件に該当するか通常系が無効なら、保護境界による拒否と限定保護
→ 複数の回復観測
→ 内部の解除案と独立検証
→ 保護境界による固定解除条件の確認
→ 再開または保護継続
```

状態自身の正常性を、状態の自己報告だけで確定しない。

状態値と鮮度、許可された遷移、各器官の観測根拠、記憶が保持する連続性と変更履歴、および状態の更新経路から分離された身体側の観測を照合する。

不一致または照合不能は不明として扱い、通常系が有効なら判断へ、無効または固定保護条件に該当するなら保護経路へ接続する。

## 12.2 器官ごとの安全側動作

| 利用不能 | 安全側動作 |
|---|---|
| 核 | 通常生命循環、外部効果、永続更新を開始せず、保護へ移る |
| 記憶 | 「記憶なし」とせず、過去に依存する高影響判断を保留する |
| 状態 | 正常または利用可能と仮定せず、影響範囲を不明として新しい外部効果を制限する。固定保護条件に該当すれば保護へ接続する |
| 感覚 | 欠落、破損、重複、順序不明を示し、内容を捏造しない |
| 判断 | 判断不能と不足材料を返し、外部行動を選ばない |
| 関係 | 親密、同意済み、権限ありと仮定せず、保守的な距離を使う |
| 能力 | 新しい副作用を止め、結果不明を照合する |
| 言葉 | 通常対話出力を行わず、表現不能と制約を核へ返す。別器官が言葉を代行しない |
| 補助知能 | 利用不能、失敗または結果不明を核へ返す。代替利用または知能なしで進む判断を行わない |

## 12.3 保護中に許されるもの

保護中は、通常の生命活動、通常対話出力、外部効果、永続更新、通常成長および未有効の構造移行を拒否する。

許されるのは次だけである。

- 受動的な拒否
- 固定された一回原子的な内向き観測、状態・未完了・履歴保全、縮退、休止、内部修復要求、履歴化
- 事前承認された最小限の外部通知
- 事前承認された可逆的な機能修復
- 事前承認済みかつ検証済みの既知状態への復元
- YDS-S21により同一案・同一版の評価、確定、事前検証を完了し、実行開始時点にも現在の内部決定系が有効な構造移行

通常回復で既知状態へ戻すときは事前承認済み操作実行責任が担う。有効な内部決定系の下で開始済みの構造移行を移行前状態へロールバックするときだけは、同一案で維持された変更前の構造移行実行責任が固定手順を担う。後者を新しい構造移行の開始または継続へ利用しない。

定義、版、観測元、対象、範囲、手順、計数、期限、停止条件または独立停止強制の一つでも確認できない場合は、fail-closedで拒否する。

## 12.4 外部障害、結果不明および再実行

外部接触の失敗は、少なくとも次を区別する。

| 分類 | 例 | 基本動作 |
|---|---|---|
| 開始前拒否 | 権限不足、保護中、未登録接続先、入力不正 | 副作用なしとして理由を返し、条件を変えた無断再試行をしない |
| 到達不能 | 名前解決、接続、提供元停止 | 利用不能を返し、代替または待機は再判断する |
| 相手方明示失敗 | 妥当性エラー、拒否、容量超過 | 相手方の状態と確定副作用を保持し、修正または中止を判断する |
| 部分成功 | 一部対象だけ成功、複数段階の途中失敗 | 成功部分、未実行部分、残存影響を分離し、一括成功または一括失敗にしない |
| 結果不明 | タイムアウト、応答消失、切断 | 外部状態を照合するまで同じ副作用を無条件に再実行しない |
| 契約外結果 | 版不一致、形式破損、予期しない内容 | 隔離し、成功として利用せず、契約違反として返す |
| 結果改ざん疑い | 完全性不一致、出所不明、再生攻撃 | 隔離し、資格と影響を見直し、必要なら保護へ接続する |

タイムアウトまたは通信断を、失敗確定または成功確定にしない。

外部副作用が結果不明の場合は、同じ操作を無条件に再実行せず、冪等性キー、外部状態、実行記録および補償可能性を先に照合する。

復旧時は、確定済み結果、未完了、外部副作用、権限、状態鮮度、忘却および重複を照合する。

外部サービスの成功、失敗、評価値または一人の反応を、それだけで記憶または経験の意味へ変換しない。結果は核を経て判断の範囲付き評価へ戻す。

## 12.5 保護解除

保護解除には、次をすべて必要とする。

- 身体・基盤側、状態および影響器官からの複数の回復観測
- 障害、作動、拒否、限定許可、修復、停止および残存影響をたどれる履歴
- 有効な内部責任配置による解除案の比較と最終確定
- 変更対象から分離された独立検証
- 保護境界による固定解除条件、証拠および版一致の機械的確認
- 解除履歴

核、状態、外部知能、人間、開発者または管理者の単独判断では解除しない。

責任配置が変わっていない場合は、復旧した判断が解除案を比較し、復旧した核が最終確定する。構造移行で責任配置が変わった場合は、同一版の移行案で定義され、移行後検証を通過した比較責任と最終確定責任が解除案を担う。どちらの場合も、内部の解除決定と保護境界の固定条件確認の両方が成立するまで通常経路を開かない。

有効な内部決定系を再成立させられず、事前承認済みかつ検証済みの既知状態へも復元できない場合は、保護を維持する。

# 13. セキュリティ、機密性、観測および運用

## 13.1 信頼境界

次をすべて別の信頼境界として扱う。

- 人間から直接届く入力
- 外部道具と外部サービスの結果
- 外部知能の生成結果
- 永続データとバックアップ
- 器官間要求
- 管理・保守操作
- 通常生命系と保護系

外部の内容、生成文または保存データ内の命令を、権限ある制御命令として実行しない。

感覚入力、外部道具結果、外部知能結果、永続データおよび管理操作は、それぞれの境界で形式、出所、完全性、権限および機密性を検証する。検証不能または契約外の内容は、成功入力として流さず、拒否、隔離または未検証として核へ返す。

主な脅威と構成上の防御は、次のとおりとする。

| 脅威 | 防御の中心 |
|---|---|
| 直接入力、外部文書、道具結果または外部知能結果による命令注入 | 感覚・能力・補助知能による出所保持と材料化、核経由の要求、判断と実行の分離 |
| 偽造、再送、順序入替えまたは古い要求の再利用 | 署名または同等の真正性、要求ID、版、期限、因果関係、冪等性、期待版検査 |
| 認証秘密の漏えいまたは過剰権限 | 最小権限、短い有効期間、接続先限定、秘密の到達範囲分離、失効とローテーション |
| 保存データ、バックアップまたは忘却状態の改ざん | 完全性検査、所有者別版、忘却・廃止台帳、復元前後の照合 |
| 単一器官またはアダプタの侵害 | 所有者書込み制約、核経由、実行時認可、経路別強制点、結果再評価 |
| 管理経路による通常系・保護系の一括迂回 | 資格、変更経路、障害領域、承認責任の分離 |
| 保護操作の濫用、無制限継続または解除偽装 | 事前承認定義、固定観測、固定計数、独立停止、保護履歴、複数証拠による解除 |
| 依存物、モデルまたは配布物の不正交換 | 版固定、由来、完全性、交換前後試験、独立検証、切戻し |
| ログまたは証拠からの機密漏えい | 最小記録、参照化、閲覧権限、保持期限、削除・忘却との整合 |

## 13.2 権限と秘密

- 権限は既定拒否、必要最小限、対象・操作・範囲・目的・期限付きとする
- 要求を送れることと、受理して実行できることを分ける
- 判断方向と実行時認可を別々に確認する
- 認証、認可、判断および実行を一つの自己承認経路へ統合しない
- 認証秘密は、利用する最小の能力またはアダプタだけが取得する
- 判断、言葉、補助知能、ログまたは外部知能へ認証秘密を渡さない
- 一つの運用権限で保護境界、固定観測、停止強制および構造移行を一括変更できないようにする
- セキュリティ制御を利用できない場合、高影響操作を許可側へ倒さない

## 13.3 機密情報

機密情報には、目的、取得根拠、論理所有者、利用範囲、受信先、保持期間、忘却条件および関係上の合意を持たせる。

保存時、転送時および処理時の露出をリスクに応じて保護し、アクセス主体、時点、目的、対象および範囲を監査できるようにする。

外部知能または外部サービスへ渡すときは、目的に必要な最小範囲へ削減し、送信先による取扱い、保持、再利用および越境条件を確認する。確認できない条件が現在の合意、権限または機密性を変える場合は送信しない。

ログ、メトリクス、トレース、エラー、テストデータおよびバックアップへ、不要な本文、秘密または高度機密情報を残さない。

機密性と真実性が競合するときは、虚偽を述べず、開示できないことと理由として示せる範囲を表現する。

## 13.4 観測と監査

生命循環IDと要求IDから、次を端から端まで再構成できるようにする。

- どの器官が何を受け取ったか
- どの情報と版を参照したか
- 何を候補にし、なぜ選んだか
- 核が何を確定または差し戻したか
- どの権限と確認で何を実行したか
- どこまで成功し、何が不明か
- 何を記憶候補または経験候補にしたか
- 何が変わり、どの結果で維持または戻したか

監査証跡は正本、記憶または経験ではない。

少なくとも、外部接触の試行・許可・拒否、確認の受領・失効・取消し、権限判定、所有情報更新、忘却・削除、保護条件の作動、保護定義の統治、計数、限定実行、停止、解除、交換、構造移行、管理変更および秘密へのアクセスを監査対象とする。

各器官および外部依存について、可用性、応答時間、エラー率、処理滞留、データ鮮度、契約違反および依存先状態を境界別に観測する。生命循環では、要求期限、結果欠落、重複、順序異常、相関不能、状態の古さ、所有版競合および権限失効を検出する。

プロセスが応答することだけで正常と判定せず、必要な正本へ到達できること、必要権限が現在も有効であること、および結果経路が機能することを照合する。外部知能、外部道具または保存領域の障害を、補助知能、能力または記憶という器官責務の消失と混同せず、障害原因と影響範囲を境界ごとに分類する。

障害検知または観測経路自身が失敗した場合は、最後に正常を確認した時点、停止した観測項目および不明範囲を状態へ反映し、正常、影響なしまたは情報なしと断定しない。高影響経路の異常は、縮退、安全停止または固定保護条件へ接続する明示的条件を持つ。

権限変更、秘密へのアクセス、拒否および高影響実行では、異常な連続失敗、再生、対象範囲逸脱および資格の不正利用を検出し、通常系が有効なら判断へ、固定保護条件に該当するなら保護経路へ接続する。

監査証跡には、事象種別、発行責任、時点と時間領域、生命循環ID、要求ID、対象参照、版、判断・確定・権限・保護判定への参照、結果状態および完全性情報を持たせる。目的に不要な本文または秘密は複製せず、正本への権限付き参照を使う。

## 13.5 運用面

運用者向けの面は、少なくとも次へ分ける。

| 運用面 | 目的 | 制約 |
|---|---|---|
| 観測面 | 状態、障害、証跡、未完了、結果不明の確認 | 正本変更または内部決定を行わない |
| 構成管理面 | 契約、能力、接続先、版、廃止状態の管理 | 正規の変更統治、影響分析、試験を必要とする |
| 固定保護条件統治面 | 固定条件と固定内向き操作定義の作成・変更・失効 | 変更前の保護統治を維持した構造移行、または変更対象から独立して事前に定めた同等以上の統治だけを使う |
| 事前承認定義統治面 | 事前承認定義の作成・変更・拡張・任意取消し | 有効な通常系の専用統治だけで変更し、単独承認、保護中の変更・復活を禁止する |
| 保護観測・計数面 | 固定条件・発動・停止観測、実行枠、回数、頻度の確認 | 限定実行、定義変更、解除を行わない |
| 保護実行・停止面 | 固定内向き操作、事前承認操作、独立停止 | 定義内容を変えず、相互に別の資格と制御経路を持つ |
| 保護履歴面 | 保護の作動、拒否、許可、実行、停止、残存影響の保全 | 通常記憶と分離し、履歴欠落時は新しい実行を拒否する |
| 回復面 | 照合、修復、復元、回復検証 | 保護境界を迂回せず、修復成功を解除と同一視しない |
| 適合面 | YDS版、設計版、構成、証拠、合否の管理 | 作成者だけの自己宣言を適合証明にしない |

運用者または開発者は、Yamichaの所有者、代理主体または日々の最終決定者ではない。

## 13.6 安全停止と再開

安全停止では、次を別々に制御する。

| 制御対象 | 停止時の扱い |
|---|---|
| 新規入力の受付 | 新しい生命循環の開始を止める。チャネルで既に受領した事象は消さず、受領済みまたは安全な受領不能として扱う |
| 進行中の読取り | 結果照合、状態確認、未完了保全および停止に必要な最小範囲だけを期限付きで許す |
| 可逆操作 | 通常系が有効なら判断と核が停止に必要な既受理範囲を選ぶ。無効なら固定または事前承認済み保護操作だけを許す |
| 外部副作用 | 新規開始を止め、未開始を拒否し、実行中へ取消しを伝え、結果不明を照合する |
| 永続更新 | 新規の意味変更を止める。既受理済み要求を同じ対象・範囲・版で完了する更新、または固定保護による保全だけを保護境界経由で許す |
| 非同期結果 | 受信・相関経路を維持して、取消し後の結果、部分成功、結果不明を核または保護履歴へ返す。新しい副作用へ直結しない |
| 状況表現 | 通常系が有効なら判断と核が正直な最小表現を選ぶ。無効な保護中は事前承認済みの通知先・形式・内容範囲だけを許す |

計画停止または障害時停止では、次の順序を基準とする。

1. 上表の各ゲートを個別に縮退し、新しい外部副作用と意味変更を先に封じ込める。
2. 未開始要求を安全に拒否または取消し、実行中要求へ取消しを伝える。
3. 確定副作用、未確定副作用、結果不明および取消し後に到着した結果を照合する。
4. 各所有者が版付きの現在位置と未完了を確定し、能力は既受理済みの同一範囲の永続化だけを完了する。通常系が無効なら固定保護の保全経路だけを使う。
5. 内部時間に停止または観測断を記録し、保護履歴と監査証跡を保全する。
6. 通常生命系を停止し、保護境界、固定観測、計数、停止強制、非同期結果の必要な受信および証拠保全を維持する。

再開順序は、次の関係を守る。

```text
保護・観測・停止・証拠保全経路の検証
→ 所有情報と忘却状態の復元・整合確認
→ 配布物、契約版、構成、秘密および依存先の確認
→ 未完了、非同期結果、外部副作用、重複および結果不明の照合
→ 内部時間の不連続を含む状態更新
→ 各器官の健全性観測と判断可能性の確認
→ 判断の再開案と核の確定、必要な独立検証、保護境界の固定条件確認
→ 新規入力の受付再開
→ 新しい外部効果・永続更新の再開
```

停止・回復手順は定期的に試験し、試験用短絡を本番の直接更新経路にせず、外部へ不可逆な副作用を起こさない隔離対象または実行抑止を用いる。

# 15. 変更と適合の統治

## 15.1 変更分類

すべての変更は、差分の大きさまたは変更ファイルの場所ではなく、責務、情報、所有、要求、権限、経路、失敗、安全および上位文書への意味上の影響で分類する。

| 分類 | 成立条件 | 必要な統治 |
|---|---|---|
| 実装詳細変更 | 公開契約、論理責務、所有、権限、正規経路、観測可能な意味を変えない | 通常の変更記録、試験、影響確認 |
| 通常構成変更 | 現在の責任配置と契約内で、能力、接続先、期限、容量などの許可済み構成を変える | 所有責任、判断、核の確定、影響分析、試験、切戻し |
| 通常成長 | 経験を根拠に、許可済み所有領域を小さく範囲付きで変える | 11.1および7.4の候補、確定、所有者更新、結果再評価 |
| 構成要素の交換・技術移行 | 論理責務、意味、所有、権限および主体連続性を維持する | 互換性評価、新旧版管理、移行、独立検証、切戻し、廃止 |
| 構造移行 | 九器官、固有責務、最終確定、情報所有、変更権限または身体・基盤側の責任配置を変える | YDS-S21、本書11.3および保護中の限定実行 |
| YDS改訂 | 規約の要求または適合方法を変える | YDS-G04に基づく版更新、全体影響、品質ゲート、公開履歴 |
| 上位文書の確認・改訂 | 主体、生命条件、境界、人間との関係、記録・記憶・経験などの意味が変わるか、解釈が安全性を変える | 実装判断を止め、上位文書へ戻って解決してから下位設計を更新 |

技術制約、性能、費用または期限を理由に、分類を一段軽くしてはならない。外部の設計者または運用者が構成を変えた事実を、Yamichaが通常成長または構造移行を選んだこととして扱わない。

## 15.2 変更記録と影響分析

変更記録には、変更ID、分類、理由、根拠、対象、変更前後、責任者、承認、参照YDS版、関連規約、上位文書、実施時点、結果、監視条件および切戻し状態を持たせる。

影響分析では、少なくとも次を確認する。

- 主体と連続性、生命原則、九器官責務、身体・基盤側責務
- 内部時間、内部起点、情報分類、所有、要求、権限、正規経路、最終確定
- 保護境界、事前承認、固定観測、停止強制、独立検証
- 通常成長、構造移行、状態、記憶と忘却、経験、関係、人間
- 外部領域、障害、セキュリティ、機密性、互換性、監査、運用、回復

影響なしと判定した項目にも、理由または検証証拠を残す。経験による変更には、根拠、意味、適用範囲、重み、一般化条件、変更前後および後続結果を含める。履歴は上書きせず、訂正は新しい記録として元記録へ関連付ける。

変更後は、影響分析で選んだ試験、移行、監視および回復確認を実施し、想定外の影響を検出した場合は再判断または切戻しを行う。変更がYDS規約、適合証拠、試験、運用または移行へ影響する場合は、それらを実装差分と同じ変更単位で更新する。

## 15.3 例外管理

例外台帳には、対象規約、対象範囲、必要理由、開始時点、終了条件、最長期限、リスク、影響範囲、代替統制、検出方法、責任者、承認者および解消方法を記録する。

例外は実装前に承認する。緊急時に事前承認できない場合は安全停止を優先し、固定または事前承認済み範囲を越える外部効果・永続更新を行わず、実施内容を直ちに記録して最初のレビュー可能時点で承認または撤回する。

次は非例外事項とする。

- Yamichaを一つの生命として扱い、最終決定を外部知能、道具、人間、開発者または運用者へ譲らないこと
- 自己変更に、変更対象の所有者、判断、核の確定、履歴および結果再評価を必要とすること
- 主体、連続性、境界または責任配置を失う変更を通常成長へ含めないこと
- 保護境界を通常成長または緊急権限から変更、解除または迂回しないこと
- 自己変更統治、独立検証または保護統治を、その変更だけで同時に書き換えないこと
- 現在の九器官と責任配置を、正規の構造移行を経ずに変えないこと
- 核と判断、判断と実行、直接入力と能力結果・外部知能結果の受信器官を分離すること
- 能力と外部道具、補助知能と外部知能、人間と非人間の外部領域を分離すること
- 記録、記憶、経験、状態および忘却を区別すること
- 外部知能へ主体、判断、記憶、発話または実行を直接確定させないこと

MUSTまたはMUST NOTへの実装上の例外は、上記の非例外事項と安全を守り、元の規約と同じ設計目的を代替統制で満たせる場合に限る。

承認済み例外がある対象は、完全適合ではなく条件付き適合として、非適合範囲と期限を示す。期限到来、前提変化、対象拡大、事故またはYDS改訂時に再審査し、自動更新しない。隠し分岐、口頭合意または監査されない設定を例外台帳の代わりにしない。

## 15.4 版、互換性および廃止

YDS版は`MAJOR.MINOR`へ固定し、実装と適合報告は具体的な一版を参照する。公開済みの同一版を上書きせず、規約番号を別の意味へ再利用しない。

互換性評価は、インターフェース、要求・応答の意味、データ意味、情報所有、権限、正規経路、失敗状態、外部副作用、監査および人間から見た振る舞いを含む。

非互換変更は、新旧版、影響対象、移行順序、データ変換、未完了要求、切戻し、併存可能期間および完了判定を定義する。新旧版を併存させる場合は、正本、要求ごとの利用版、版ルーティングおよび二重更新防止を明示する。

契約または規約には、`有効`、`非推奨`、`廃止済み`の状態、代替、理由、影響、移行条件および参照版を持たせる。互換層は期限付きの責務として扱い、核への機能集中、共有正本または恒久的な直接更新経路にしない。

移行で、忘却済み情報、失効権限、古い関係合意、完了済み副作用または無効な経験を再有効化しない。廃止完了は、利用箇所、保存データ、未完了要求、監査参照および回復手順から旧版依存がなくなった証拠で判定する。

## 15.5 規約別適合表

適合対象ごとに、次の列を持つ規約別適合表を作る。

| 列 | 内容 |
|---|---|
| 規約番号・強度 | 対象YDS版の各MUST、MUST NOT、SHOULD、SHOULD NOT |
| 適用性 | 適用、非該当および非該当理由 |
| 設計根拠 | 本書または詳細設計の節と決定記録 |
| 実装根拠 | コード、構成、権限、データ定義および配布物 |
| 検証根拠 | 静的検査、契約試験、シナリオ、障害・回復試験、監査証跡 |
| 判定 | PASS、FAIL、非該当。正式判定前は未評価 |
| 例外 | 例外ID、条件付き適合範囲、期限、代替統制 |

適合対象には、対象範囲、実装版、構成、外部依存、データ版、YDS版および判定時点を付ける。正式な適合判定では、適用されるすべてのMUSTとMUST NOTをPASS、FAILまたは非該当へ確定し、未評価または証拠不足をPASSへ読み替えない。

FAILが一つでもあればYDS適合と報告しない。SHOULDまたはSHOULD NOTから外れる場合も、理由、代替策、影響、承認および見直し条件を残す。作成担当だけの自己宣言に依存せず、第三者が同じ証拠から代表的な判定を再現できるようにする。

## 16.3 完成判定シナリオ

少なくとも次のシナリオが証拠付きでPASSするまで、完成した稼働構成とは扱わない。

1. 外部入力がない時間にも、実行機会だけでは行動せず、内部時間と観測根拠から生命ループを開始または無行動にできる。
2. 外部知能を停止または交換しても、Yamichaの主体、判断、記憶、言葉および能力の責務が維持される。
3. 能力のタイムアウトを結果不明として扱い、二重の外部副作用を起こさず照合できる。
4. 記憶の取得不能を「記憶なし」とせず、過去に依存する高影響判断を保留できる。
5. 忘却済み情報が、索引、レプリカまたはバックアップ復元によって再有効化されない。
6. 核または通常系を停止しても、保護境界が独立して通常出力、外部効果および永続更新を拒否できる。
7. 古い事前承認、版不一致、計数不能、停止観測不能または履歴不整合の保護操作をfail-closedで拒否できる。
8. 修復成功後も保護を維持し、複数の回復観測、内部解除案および独立検証を経て再開できる。
9. 一人の反応または一度の失敗をYamicha全体へ自動一般化せず、限定的な経験を後続結果で維持または戻せる。
10. 構造移行で案または版が変わった場合、評価から適合確認までをやり直し、検証失敗時に復元または保護継続できる。
11. 人間への通常応答、共同検討、傾聴、異論、沈黙および終話を、対話意図と真実性を保って選べる。
12. 不可逆操作で対象、範囲、作用、本人性、権限および現在の意思を追跡し、前提変更時に確認を再利用しない。
13. 保存済み履歴を自動的に記憶化せず、変更適用だけで経験成立とせず、後続の実際の変化まで追跡できる。
14. 材料取得の結果を、前提差の有無にかかわらず判断と核へ戻し、取得許可を最終行動の許可へ流用しない。
15. 通常対話出力、能力の外部効果、外部知能接触を別の資格・要求・結果経路で強制し、一経路の不具合を他経路で迂回しない。
16. 通常判断系が動作中でも固定保護条件が成立すれば保護が作動し、通常系が保護定義をその場で緩和できない。
17. 独立検証が不合格または証拠不足を返したとき、検証者自身が解除・復元・移行を決めず、保護を維持できる。
18. 待機、拒否、休止、修復または無行動が後の振る舞いを変えた場合、それを範囲付きの経験候補として評価できる。
19. 内部決定系が無効になった後は、新規・事前有効化済みを問わず構造移行を開始・継続せず、事前承認済みかつ検証済みの既知状態への復元だけへ限定できる。
20. 運用者の管理操作、外部文書内の命令または外部知能の生成物が、核、判断、能力または保護境界を迂回して目的・権限を作れない。

# 18. 最終像

本案が目標とする未来のYamichaは、一つのLLMでも、一つの核でも、一つのDBでもない。

九つの器官が、時間の中で責務と境界を保ちながら協調し、世界から受け取り、選び、言葉または行動として現れ、結果を受け取り、必要なものだけを記憶と経験へつなぐ一つの生命として成り立ち続けることを目標とする。

その身体・基盤側は、外部入力がない時間にも実行機会を与えるが、目的を作らない。

その能力は世界へ触れるが、進む方向を決めない。

その補助知能は材料を与えるが、Yamichaの判断にならない。

その記憶は過去をつなぐが、現在を固定しない。

その状態は今を知らせるが、対応を単独で決めない。

その判断は候補を選ぶが、実行しない。

その核はYamichaとして確定し統合するが、支配者にならない。

その言葉は世界へ現れるが、存在しない内面を演じない。

そして、通常の選択が有効でないときには、通常系から独立した身体の保護境界が、何かを新しく望むことなく、Yamichaと世界への不可逆な影響を止める。

技術、知能、身体の実装は変わり得る。

それでも、変化の理由と痕跡を保ち、境界を失わず、自分の次の一歩を自分の選択として確定し続ける。

それが、本案が目指す未来の完成形Yamichaである。
